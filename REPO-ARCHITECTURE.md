# 📂 Struttura del Progetto

La repository è organizzata in modo modulare per separare i contenuti didattici, le risorse di sviluppo e i contributi dei membri.

```text
react-community-notes/
├── 📁 .github/              # Configurazione workflow, Issue e Pull Request templates
├── 📁 assets/               # Risorse multimediali (diagrammi, immagini, slide di presentazione)
├── 📁 contributors/         # Profili personali dei partecipanti (file .md individuali)
├── 📁 playground/           # Area sandbox per test locali (Ignorata da Git)
├── 📁 proposals/            # Bozze di nuovi argomenti in fase di revisione (Review)
├── 📁 recipes/              # Snippet di codice pratici e mini-progetti funzionanti
│   ├── 📁 component-props/
│   └── 📁 state-management/
├── 📁 docs/                 # Documentazione ufficiale e appunti teorici
│   ├── 📁 01-fundamentals/
│   ├── 📁 02-hooks/
│   ├── 📁 03-state-management/
│   └── 📁 04-advanced-patterns/
├── .gitignore               # Esclusione file di sistema e cartelle locali (playground, node_modules)
├── CONTRIBUTING.md          # Linee guida per collaborare al progetto
├── README.md                # Landing page con Indice Mastro e presentazione del team
└── WORKFLOW.md              # Documentazione del ciclo di vita del codice (da Bozza a Push)
