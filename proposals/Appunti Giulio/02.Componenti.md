# 🧱 Guida allo Sviluppo a Componenti con React

In React, l’interfaccia utente (UI) non viene concepita come un unico file HTML massiccio, ma come un insieme di pezzi indipendenti, isolati e riutilizzabili chiamati **Componenti**. Possiamo immaginarli come i mattoncini LEGO: ogni pezzo ha una forma e una funzione specifica, ma l'unione di più pezzi permette di costruire strutture estremamente complesse.

---

## 📜 1. Anatomia e Regole d'Oro di un Componente

Per convenzione tecnica e strutturale, ogni componente React deve seguire delle regole rigide per essere isolato e riconosciuto correttamente dal motore di rendering:

1. **PascalCase (Nome in Maiuscolo):** Il nome della funzione e del file deve tassativamente iniziare con la lettera maiuscola (es. `Card.jsx`, `Button.jsx`). React usa questa convenzione per distinguere i componenti personalizzati dai tag HTML standard (che sono sempre minuscoli come `div`, `main`, `p`).
2. **Estensione `.jsx`:** Segnala che il file contiene codice JavaScript misto a sintassi XML (HTML-like).
3. **Unicità del File:** Per garantire pulizia e modularità, ogni file ospita solitamente un singolo componente principale, esportato come default.

### 💻 Snippet: Creazione e Utilizzo di un Componente

**Creazione del Mattoncino (`src/components/Card.jsx`):**

```jsx
// 1. Definizione della funzione in PascalCase
function Card() {
  const titolo = "Prodotto Speciale";

  // 2. Il componente deve restituire (return) del codice JSX
  return (
    <div className="card-container">
      <h2>{titolo}</h2>
      <p>Questa è una descrizione del componente Card.</p>
      <button>Acquista ora</button>
    </div>
  );
}

// 3. Esportazione di default per renderlo riutilizzabile
export default Card;

```

**Assembraggio nel Regista (`src/App.jsx`):**

```jsx
// Importiamo il componente Card dal suo file dedicato
import Card from './components/Card.jsx';

function App() {
  return (
    <main>
      <h1>Il mio Store</h1>
      
      {/* Utilizzo del componente come tag auto-chiudente */}
      <Card />
      <Card />
      <Card />
    </main>
  );
}

export default App;

```

## ⚙️ 2. Separazione delle Responsabilità: Componenti vs Utility

Un’architettura professionale scinde nettamente la **logica di calcolo** (puro JavaScript) dalla **logica di visualizzazione** (struttura JSX).

* `src/components/` $\rightarrow$ File `.jsx` dedicati all'interfaccia.
* `src/utility/` $\rightarrow$ File `.js` dedicati a funzioni matematiche, formattazioni o chiamate API.

### 💻 Snippet: Integrazione di Funzioni pure e Rendering Condizionale

**La Funzione Utility (`src/utility/random.js`):**

```javascript
// Esportazione nominata (Named Export) di una funzione di calcolo
export const getRandomInt = (min, max) => {
  return Math.floor(Math.random() * (max - min + 1)) + min;
};

```

**Il Componente come Regista Dinamico (`src/App.jsx`):**

```jsx
import { getRandomInt } from './utility/random.js';
import Header1 from './components/Header1.jsx';
import Header2 from './components/Header2.jsx';
import Header3 from './components/Header3.jsx';

function App() {
  const num = getRandomInt(0, 99);
  let contenuto = null;

  // Logica per assegnare dinamicamente un intero componente a una variabile
  if (num < 33) {
    contenuto = <Header1 />;
  } else if (num < 66) {
    contenuto = <Header2 />;
  } else {
    contenuto = <Header3 />;
  }

  // React renderizzerà direttamente il componente salvato nella variabile
  return <main>{contenuto}</main>;
}

export default App;

```

## 🏗️ 3. Architettura del Layout: Macro, Micro e Nesting

I componenti seguono una struttura gerarchica ad albero (Relazione Padre-Figlio) e si dividono in due categorie principali:

1. **Macro Layout (La Cornice):** Definisce lo scheletro e le zone di ingombro strutturali della pagina (es. `Navbar.jsx`, `Footer.jsx`, `MainLayout.jsx`).
2. **Micro Layout (I Mattoncini):** Componenti atomici, altamente riutilizzabili e indipendenti dal contesto (es. `Button.jsx`, `Badge.jsx`).

Per creare dei veri e propri modelli grafici (Template), React mette a disposizione la proprietà speciale **`props.children`**. Essa consente di creare un componente "involucro" (Wrapper) capace di ospitare al suo interno qualsiasi codice o tag inserito all'esterno.

### 💻 Snippet: Nesting e utilizzo di `children`

**Creazione del Wrapper di Layout (`src/components/layout/PageWrapper.jsx`):**

```jsx
import Navbar from '../macro/Navbar.jsx';
import Footer from '../macro/Footer.jsx';

function PageWrapper({ children }) {
  return (
    <div className="page-container">
      <Navbar />
      
      {/* Il "segnaposto" in cui verrà iniettato il contenuto dinamico delle sottopagine */}
      <main className="content">
        {children} 
      </main>
      
      <Footer />
    </div>
  );
}

export default PageWrapper;

```

**Applicazione del Layout in `App.jsx`:**

```jsx
import PageWrapper from './components/layout/PageWrapper.jsx';
import Card from './components/Card.jsx';

function App() {
  return (
    <PageWrapper>
      <h2>Benvenuto nella Homepage</h2>
      <p>I blocchi inseriti qui sotto prenderanno il posto di "children".</p>
      <Card />
    </PageWrapper>
  );
}

```

## 🎭 4. La Sintassi JSX: Interpolazione ed Espressioni `{ }`

Le parentesi graffe `{ }` rappresentano una "porta di uscita" dalla modalità HTML-like per entrare momentaneamente in modalità JavaScript puro. Qualsiasi espressione valida inserita nelle graffe viene valutata, calcolata e infine convertita in stringa per il DOM.

* **Cosa PUOI inserire:** Variabili (`{titolo}`), operazioni matematiche (`{5 + 5}`), chiamate a funzioni (`{getRandomInt()}`), operatori ternari (`{isOnline ? '🟢' : '🔴'}`), o metodi nativi (`{testo.toUpperCase()}`).
* **Cosa NON puoi inserire:** Blocchi di codice non immediati o privi di valore di ritorno, come cicli `for` estesi o strutture `if / else` complete.
* **Comportamento con i Booleani, Null e Undefined:** Se un'espressione restituisce `true`, `false`, `null` o `undefined`, React **non mostra nulla a schermo**. Questa caratteristica è fondamentale per gestire il rendering condizionale.

## 🎨 5. Gestione degli Attributi e dello Stile in JSX

### Attributi Statici vs Dinamici

* **Statici:** Si utilizzano le virgolette classiche (`id="main"`). Il valore è un testo immutabile.
* **Dinamici:** Si utilizzano le parentesi graffe (`id={idDinamico}`). **Non si devono mischiare graffe e virgolette** (`id="{id}"` è un errore sintattico).
* **Il caso `class`:** In JSX non si può usare la parola `class` poiché è riservata in JavaScript per la creazione di classi ad oggetti. Si usa al suo posto **`className`**.

### Stile Inline e Oggetti

In React lo stile inline si esprime tramite un **oggetto JavaScript** scritto con la sintassi delle **doppie graffe `{{ }}**`. Le proprietà CSS con trattino vengono convertite in **camelCase**:

```jsx
<div style={{ backgroundColor: 'blue', fontSize: '16px', marginTop: '10px' }}>
  Contenuto Strizzato
</div>

```

### CSS Modules: Isolamento Totale dello Stile

Per evitare che classi CSS con lo stesso nome entrino in conflitto tra componenti diversi, si utilizzano i **CSS Modules** (file con estensione `.module.css`). Vite trasforma queste classi in oggetti JavaScript e applica loro un hash univoco di sicurezza in fase di compilazione (es. `.card` diventa `.Card_card__x1y2z`).

### 💻 Snippet: Gestione Avanzata degli Attributi e CSS Modules

**Il File Stili (`src/components/Card.module.css`):**

```css
.card-container {
  border: 1px solid #ccc;
  border-radius: 8px;
}
.active {
  border-color: green;
}

```

**Il Componente JSX (`src/components/Card.jsx`):**

```jsx
// Importiamo il CSS Module come un oggetto JavaScript nominato 'styles'
import styles from './Card.module.css';

function Card() {
  const isActive = true;
  
  // Utilizzo di un array e .join(' ') per concatenare classi condizionali
  const classiAttive = [
    styles['card-container'], // Accesso a parentesi quadre per via del trattino
    isActive ? styles.active : ''
  ].join(' ');

  return (
    <div className={classiAttive}>
      <h2 style={{ color: 'darkgray' }}>Card Modulare</h2>
    </div>
  );
}

export default Card;

```

## 🏁 6. Capitolo Riassuntivo e Schematico

### 🗺️ Mappa delle Convenzioni di Scrittura (Casing)

* `MioComponente.jsx` $\rightarrow$ **PascalCase**: Obbligatorio per File e Funzioni Componente in React.
* `miaVariabile` / `miaFunzione()` $\rightarrow$ **camelCase**: Standard per variabili, funzioni normali e stili inline in JSX.
* `stile['card-container']` $\rightarrow$ **Sintassi a Quadre**: Necessaria nei CSS Modules se il nome della classe contiene trattini.

### 📐 La Gerarchia dell'Architettura (Atomic Design)

$$\text{Atoms (Micro: Pulsante)} \longrightarrow \text{Molecules (Micro: Barra di Ricerca)} \longrightarrow \text{Organisms (Macro: Header)} \longrightarrow \text{Templates (Macro: Layout App)}$$

### 📝 Tabella di Confronto Rapido: Logica e Visualizzazione

| Concetto | Sintassi / Strumento | Campo di Applicazione / Regola d'oro |
| --- | --- | --- |
| **Componente UI** | File `.jsx` | Nome in **PascalCase**. Deve restituire un solo tag radice o un Fragment `<>...</>`. |
| **Funzione Utility** | File `.js` | Contiene solo logica pura (no codice HTML/JSX al suo interno). |
| **Interpolazione** | `{ espressione }` | Esegue JavaScript nel codice HTML. Non accetta cicli `for` o blocchi `if/else`. |
| **Attributo statico** | `className="box"` | Testo fisso. Sostituisce il vecchio `class` di HTML per non fare conflitto con JS. |
| **Attributo dinamico** | `src={percorsoFoto}` | Prende il valore da variabili o costanti. **Vietate** le virgolette intorno alle graffe. |
| **Stile Inline** | `style={{ property: 'val' }}` | Richiede l'oggetto JS. Le proprietà CSS cambiano da kebab-case a **camelCase**. |
| **CSS Modules** | `import s from './F.module.css'` | Isola gli stili a livello locale generando hash automatici anti-conflitto. |
| **Template Dynamic** | `{ children }` | Parola chiave per iniettare contenuti dinamici all'interno di una cornice (Wrapper). |
