# 🔄 Guida al Rendering Dinamico e alla Gestione delle Liste in React

In JavaScript tradizionale (imperativo), manipolare il DOM per creare liste di elementi richiede cicli complessi e manipolazioni manuali. In React si adotta invece un approccio **dichiarativo**: si parte da un array di dati grezzi e lo si trasforma direttamente in un array di elementi JSX.

---

## 🏎️ 1. Dal Ciclo `for` al Metodo `.map()`

Anche se un classico ciclo `for` con l'utilizzo di `.push()` in un array esterno è logicamente valido, in React risulta prolisso e inefficiente perché costringe a scrivere la logica fuori dal blocco di rendering principale.

### 🛑 Approccio Imperativo (Sconsigliato)

```jsx
function ListaImperativa() {
  const elementi = ['Elemento 1', 'Elemento 2', 'Elemento 3'];
  let listaJsx = [];

  for (let i = 0; i < elementi.length; i++) {
    const currentEl = elementi[i];
    listaJsx.push(<li key={i}>{currentEl}</li>);
  }

  return <ul>{listaJsx}</ul>;
}

```

### 🚀 Approccio Dichiarativo con `.map()` (Standard React)

Il metodo `.map()` trasforma ogni singolo dato dell'array in un elemento HTML/JSX. Può essere inserito direttamente all'interno del `return` grazie all'interpolazione con le parentesi graffe `{ }`.

```jsx
function Navbar() {
  const links = ["Characters", "Comics", "Movies", "TV", "Games"];

  return (
    <nav className="navbar">
      <ul className="navbar-nav">
        {/* Iterazione diretta e pulita */}
        {links.map((link, index) => (
          <li className="nav-item" key={index}>
            <a className="nav-link" href="#">{link}</a>
          </li>
        ))}
      </ul>
    </nav>
  );
}

```

### 🚀 Ottimizzazione: Allocazione in Memoria vs Definizione

Quando generiamo una lista di elementi con un bottone di azione (es. rimuovi), dobbiamo evitare di definire funzioni anonime direttamente dentro il ciclo `.map()`.

* **Anti-pattern (Spreco di memoria):** Definire `onClick={() => handleRemove(id)}` dentro il `.map()` costringe il Garbage Collector a allocare una nuova istanza di funzione in memoria per ciascun elemento presente nella lista, a ogni singolo ciclo di rendering.
* **Soluzione Ottimizzata:** Definire un'unica funzione centralizzata nel corpo principale del componente e parametrizzarla passando l'ID dell'elemento.

```jsx
import { useState } from "react";

function ListaUtenti() {
  const [userList, setUserList] = useState([
    { id: "u1", name: "Marco", address: "Via Roma 1" },
    { id: "u2", name: "Luigi", address: "Via Milano 10" }
  ]);

  // --- LOGICA CENTRALIZZATA ---
  // Creata una volta sola in memoria, a prescindere dal numero di elementi in lista
  const removeUserById = (id, event) => {
    // ✅ Immutabilità degli array tramite .filter()
    const newUserList = userList.filter(user => user.id !== id);
    setUserList(newUserList);
  };

  return (
    <div>
      <h1>Elenco Utenti</h1>
      <ul>
        {userList.map((user) => {
          const userId = user.id; // Salviamo l'ID in una costante per chiarezza

          return (
            // ✅ Uso di una KEY univoca e stabile (MAI l'index del ciclo)
            <li key={userId}>
              {`${user.name} - ${user.address}`}
              
              {/* Passiamo l'id alla funzione centralizzata tramite la referenza */}
              <button onClick={(event) => removeUserById(userId, event)}>
                Rimuovi
              </button>
            </li>
          );
        })}
      </ul>
    </div>
  );
}

```

### 🔑 I Tre Pilastri delle Liste in React

1. **Rendering Dichiarativo con `.map()`:** Prende l'array di dati grezzi e lo mappa restituendo un array corrispondente di nodi JSX pronti per lo schermo.
2. **Immutabilità con `.filter()`:** Per rimuovere un elemento dall'array di stato non usiamo metodi distruttivi come `.splice()`. Usiamo `.filter()`, che genera un array nuovo di zecca escludendo l'elemento desiderato, salvaguardando il tracciamento dei rendering di React.
3. **L'importanza Cruciale delle `key` Stabili:** La `key` permette a React di identificare univocamente ogni nodo all'interno del Virtual DOM. **Non usare mai l'indice (`index`) del ciclo come chiave**. Se la lista subisce un riordinamento o un'eliminazione, gli indici slitterebbero posizionalmente, causando gravi bug grafici (es. il testo inserito in un campo `<input>` rimarrebbe ancorato alla riga precedente anziché seguire l'elemento rimosso o spostato). Utilizza sempre ID univoci stabili (es. estratti dal database o generati all'origine tramite `crypto.randomUUID()`).

## 🔑 2. La Regola d'Oro della Prop `key`: ID vs Index

Quando React renderizza una lista di elementi, richiede obbligatoriamente un attributo speciale chiamato `key` per tenere traccia dell'identità di ogni nodo nel DOM.

* **Utilizzare un ID Univoco (Best Practice):** Se i dati provengono da un database o possiedono un ID fisso (`key={item.id}`), React usa questa informazione come un'impronta digitale. Se la lista viene riordinata, filtrata o modificata, React sposta solo l'elemento interessato senza ricaricare l'intera lista.
* **Utilizzare l'Indice del ciclo (`index` - Da usare con cautela):** Usare `key={index}` va bene solo se la lista è **statica** (non cambia mai ordine o numero di elementi). Se la lista è dinamica (es. eliminazione di un elemento), gli indici cambiano e React potrebbe confondersi, generando bug grafici nel rendering del componente di cui è stato modificato l'indice in array.

## 📦 3. Iterazione Avanzata su Array di Oggetti

Nello sviluppo reale, i dati sono quasi sempre strutturati come array di oggetti. Per mantenere il codice pulito, si ricorre spesso al **Destructuring** direttamente nei parametri del ciclo `.map()`.

```jsx
function BlueBar() {
  const blueMenu = [
    { id: 'm1', text: "Digital Comics", url: "/img/comics.png" },
    { id: 'm2', text: "DC Merchandise", url: "/img/merch.png" },
    { id: 'm3', text: "Subscription", url: "/img/sub.png" }
  ];

  return (
    <section className="bg-primary py-4">
      <ul className="d-flex justify-content-center list-unstyled">
        {/* Destructuring delle proprietà dell'oggetto direttamente nel ciclo */}
        {blueMenu.map(({ id, text, url }) => (
          <li key={id} className="d-flex align-items-center mx-3">
            <img src={url} alt={text} style={{ width: '45px' }} />
            <span className="text-white ms-2 text-uppercase">{text}</span>
          </li>
        ))}
      </ul>
    </section>
  );
}

export default BlueBar;
```

## 🎭 4. Rendering Condizionale e Gestione della Visibilità

Esistono diverse tecniche per mostrare o nascondere gli elementi di una lista o per applicare classi CSS dinamiche (come la classe `attiva` sul link corrente).

### A. Gestione delle Classi Dinamiche (Ternario vs Short-Circuit)

Per aggiungere classi in base a uno stato booleano (es. `current: true`), l'operatore ternario è lo standard assoluto in React.

```jsx
// Metodo con Operatore Ternario (Compatto e Dichiarativo)
<a className={current ? "nav-link attiva" : "nav-link"} href={url}>
  {text}
</a>

// Metodo con Cortocircuito Logico && (Se non serve una classe alternativa)
<a className={`nav-link ${current && "attiva"}`} href={url}>
  {text}
</a>

```

### B. Gestione della Visibilità degli Elementi

Se un elemento ha una proprietà `hidden: true`, possiamo decidere di nasconderlo tramite CSS o di escluderlo del tutto dal codice della pagina.

```jsx
// Opzione 1: Nascondere tramite CSS (L'elemento rimane nel DOM)
<li className={hidden ? "d-none" : "d-block"}>...</li>

// Opzione 2: Cortocircuito && (L'elemento viene rimosso dal DOM se nascosto)
{ !hidden && <li>...</li> }

// Opzione 3: Filtraggio preventivo con .filter() prima del .map()
{links
  .filter(link => !link.hidden) // Passano solo i link visibili
  .map(link => (
    <li key={link.id}>{link.text}</li>
  ))
}

```

## 🏗️ 5. Il Pattern "Prepare & Render" e Griglie Responsive

Nelle interfacce complesse, è consigliabile separare la logica di trasformazione dei dati dal template finale del componente. Questo approccio pulisce il blocco di `return` principale e rende il codice altamente scalabile e responsivo.

```jsx
function BannerGrid() {
  const banners = [
    { id: 1, title: "Digital Comics", img: "/img/comics.png" },
    { id: 2, title: "Merchandise", img: "/img/merch.png" },
    { id: 3, title: "Shop Locator", img: "/img/shop.png" }
  ];

  // 1. FASE DI PREPARAZIONE (Trasformazione dei dati)
  const bannerJsx = banners.map(({ id, title, img }) => {
    const colClasses = "col d-flex align-items-center justify-content-center py-2";
    
    return (
      <div key={id} className={colClasses}>
        <img src={img} alt={title} style={{ width: '40px' }} />
        <span className="ps-3 fw-bold">{title.toUpperCase()}</span>
      </div>
    );
  });

  // 2. FASE DI RENDERING (Template pulito e griglia Bootstrap)
  return (
    <div className="bg-dark text-white py-5">
      <div className="container">
        {/* Griglia intelligente: 1 colonna su mobile, 3 su desktop */}
        <div className="row row-cols-1 row-cols-md-3">
          {bannerJsx}
        </div>
      </div>
    </div>
  );
}

export default BannerGrid;

```

## 🏁 6. Capitolo Riassuntivo e Schematico

### 📐 Tabella Comparativa: Tecniche di Controllo Visibilità

| Tecnica | Sintassi Esemplificativa | Stato nel DOM | Utilizzo Consigliato |
| --- | --- | --- | --- |
| **Classi Condizionali** | `className={active ? 'active' : ''}` | Presente | Cambi di stile immediati (es. tab attive, hover di stato). |
| **CSS Utilities** | `className={hidden ? 'd-none' : ''}` | Presente (Invisibile) | Toggle molto frequenti o elementi legati a transizioni animate. |
| **Cortocircuito `&&**` | `!hidden && <Elemento/>` | Assente (Rimosso) | Messaggi di errore dinamici, popup condizionali o permessi utente. |
| **Chaining `.filter().map()**` | `arr.filter(f).map(m)` | Assente (Rimosso) | Grandi moli di dati strutturati per evitare calcoli inutili su nodi nascosti. |

### 🧠 Regole Mnemoniche per i Cicli in React

* 💡 **Niente Cicli Imperativi nel `return**`: Non usare mai cicli `for` o `while` dentro la struttura JSX. Usa esclusivamente espressioni dichiarative come `.map()`.
* 💡 **Le parentesi tonde nel `.map()**`: Quando apri una freccia nel map, l'uso delle parentesi tonde `() => ( ... )` esegue un **return implicito** del JSX. Se usi le graffe `() => { ... }`, devi scrivere esplicitamente la parola chiave `return`.
* 💡 **L'immutabilità della `key**`: La `key` deve essere stabile, univoca e assegnata al **tag più esterno** restituito dal ciclo (il tag "padre" dell'iterazione).
