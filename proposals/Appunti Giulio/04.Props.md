## 🔌 1. Il Passaggio delle Props

Nel componente principale (Padre), richiamiamo il mattoncino della nostra interfaccia passandogli dei dati specifici per configurare ogni singola stanza dell'abitazione.

### 💻 Esempio di Invocazione (Componente Padre)

```jsx
// src/App.jsx
import PannelloStanza from './components/PannelloStanza.jsx';

function App() {
  return (
    <main className="dashboard">
      <h1>Smart Home Control Panel</h1>
      
      {/* Caso 1: Passiamo tutte le informazioni richieste */}
      <PannelloStanza 
        stanza="Soggiorno" 
        dispositivoAttivo="Condizionatore" 
      />

      {/* Caso 2: Passiamo solo la prop obbligatoria */}
      <PannelloStanza 
        stanza="Corridoio" 
      />
    </main>
  );
}

export default App;

```

## 🏗️ 2. Destructuring, Default Props e Ridenominazione (Alias)

All'interno del componente Figlio, destrutturiamo l'oggetto delle props in arrivo. Impostiamo una stanza di default nel caso in cui il Padre non la specifichi e rinominiamo una variabile per renderla semanticamente più pulita nel nostro codice.

### 💻 Snippet: Destructuring Avanzato

```jsx
// src/components/PannelloStanza.jsx

// Estraiamo 'stanza', lo rinominiamo in 'nomeStanza' e diamo un fallback di default.
// Estraiamo 'dispositivoAttivo' direttamente senza rinominarlo.
function PannelloStanza({ stanza: nomeStanza = 'Stanza Generica', dispositivoAttivo }) {
  
  // NOTA: La variabile originaria 'stanza' non è più accessibile, usiamo 'nomeStanza'
  return (
    <div className="room-card">
      <h2>{nomeStanza}</h2>
      <p>Dispositivo in funzione: {dispositivoAttivo}</p>
    </div>
  );
}

export default PannelloStanza;

```

## 🧐 3. Risoluzione dei Nodi Vuoti con il Cortocircuito (`&&`)

Nel secondo esempio di chiamata del capitolo 1, il `dispositivoAttivo` non viene configurato. Il codice precedente genererebbe un tag `<p>Dispositivo in funzione: </p>` parzialmente vuoto, che altera l'altezza delle card e sporca l'interfaccia.

Risolviamo applicando il cortocircuito logico: il tag si materializza nel DOM solo se la stringa esiste.

```jsx
// src/components/PannelloStanza.jsx

function PannelloStanza({ stanza: nomeStanza = 'Stanza Generica', dispositivoAttivo }) {
  return (
    <div className="room-card">
      <h2>{nomeStanza}</h2>
      
      {/* Il tag <p> e il suo testo compaiono SOLO se viene passato un dispositivo */}
      {dispositivoAttivo && (
        <p>Dispositivo in funzione: {dispositivoAttivo}</p>
      )}
    </div>
  );
}

export default PannelloStanza;

```

## 🥗 4. Rendering di Liste da Oggetti e Operatore Ternario

Un sensore della nostra casa intelligente restituisce lo stato dei vari sistemi di sicurezza sotto forma di oggetto con coppie chiave-valore booleane. Lo trasformiamo in un array ciclabile tramite `Object.entries()` e convertiamo i booleani in icone tramite l'operatore ternario.

### 💻 Snippet: Trasformazione e Operatore Ternario `? :`

```jsx
// src/components/StatusSicurezza.jsx

function StatusSicurezza() {
  // Dati di telemetria ricevuti sotto forma di oggetto
  const reportSistemi = {
    allarmeAntifurto: true,
    rilevatoreFumo: false,
    sensoreAllagamento: false
  };

  // 1. Convertiamo l'oggetto in un array: [['allarmeAntifurto', true], ...]
  const listaSistemiJsx = Object.entries(reportSistemi).map(([sistema, attivo]) => {
    return (
      // Utilizziamo il nome del sistema (chiave dell'oggetto) come key univoca stabile
      <li key={sistema} className="system-item">
        <span className="system-name">{sistema}:</span>
        {/* Il ternario decide la dicitura visiva in base al booleano */}
        {attivo ? ' Attivo 🛡️' : ' Spento 💤'}
      </li>
    );
  });

  return (
    <div className="security-panel">
      <h3>Diagnostica Sicurezza:</h3>
      <ul>{listaSistemiJsx}</ul>
    </div>
  );
}

export default StatusSicurezza;

```

## 🧩 5. Props Avanzate: `children` e Slots Complessi

Evolviamo l'applicazione creando dei layout modulari per la nostra interfaccia di controllo.

### A. La Special Prop `children`

Creiamo una cornice grafica generica per i widget della dashboard. Tutto ciò che inseriamo tra i tag di apertura e chiusura verrà stampato automaticamente al centro della card.

```jsx
// src/components/WidgetWrapper.jsx (Il contenitore generico)
function WidgetWrapper({ children }) {
  return <div className="widget-box-shadow-theme">{children}</div>;
}

// Come viene utilizzato nel file principale Padre
function App() {
  return (
    <WidgetWrapper>
      <h3>Consumo Energetico</h3>
      <div className="gauge">2.4 kW/h</div>
      <button className="btn-details">Vedi Grafico</button>
    </WidgetWrapper>
  );
}

```

### B. Componenti passati come Props Personalizzate (Slots)

Se vogliamo creare una barra di controllo avanzata che richiede un'icona o un pulsante di azione specifico posizionato a destra e una serie di informazioni a sinistra, passiamo direttamente i blocchi JSX tramite props dedicate.

```jsx
// src/components/BarraControllo.jsx (Componente strutturale a slot)
function BarraControllo({ areaInfo, areaAzione }) {
  return (
    <div className="control-bar-layout">
      <div className="left-side-info">{areaInfo}</div>
      <div className="right-side-action">{areaAzione}</div>
    </div>
  );
}

// Utilizzo nel componente Padre per configurare un controllo luci
function App() {
  return (
    <BarraControllo 
      areaInfo={
        <>
          <h4>Illuminazione Esterna</h4>
          <span className="status">Attiva dal tramonto</span>
        </>
      }
      areaAzione={<button className="btn-toggle">Spegni Ora</button>}
    />
  );
}

```

## 🏁 6. Capitolo Riassuntivo e Schematico

### 📊 Tabella Comparativa delle Tecniche di Iniezione JSX

| Tecnica | Esempio Sintetico (Smart Home) | Meccanismo | Scenario d'Uso Corretto |
| --- | --- | --- | --- |
| **Default Props** | `({ stanza = 'Corridoio' })` | Se il padre non passa il dato, usa la stringa di fallback. | Proprietà fondamentali per evitare stringhe spezzate o errori di rendering. |
| **Cortocircuito `&&**` | `dispositivo && <p>{dispositivo}</p>` | Cancella il tag dal DOM se la variabile è falsa o vuota. | Evitare tag orfani o spazi bianchi strutturali non voluti nel layout. |
| **Operatore Ternario** | `attivo ? 'On 🟢' : 'Off 🔴'` | Genera un'alternativa di testo o di tag secco (If / Else). | Cambiamenti di stato binari veloci scritti direttamente in linea nel JSX. |
| **Prop `children**` | `<Widget> <Contenuto/> </Widget>` | Cattura e inietta il codice racchiuso tra i tag di apertura/chiusura. | Componenti contenitore grafici (Box, Modali, sezioni della griglia). |
| **Props Nominate (Slots)** | `<Barra areaAzione="{<Bottone"/>} />` | Assegna blocchi di interfaccia a variabili specifiche ben posizionate. | Layout flessibili multi-zona (es. Sidebar + Main, Header con slot per icone). |

### 🧠 Regole Mnemoniche del Flusso Dati

* 💡 **Le Props sono Sacre**: Un componente non può modificare le props che riceve. Se `PannelloStanza` riceve `stanza="Soggiorno"`, non può sovrascrivere internamente quel valore. Le props sono in sola lettura.
* 💡 **L'estrazione degli oggetti**: `Object.entries(oggetto)` converte le coppie in un array leggibile da `.map()`. Ricorda di fare destructuring della coppia `[chiave, valore]` subito all'inizio del ciclo per avere un codice pulito.
* 💡 **Identificativi nelle Key**: Quando cicli le proprietà estrante da un oggetto, usa il nome della chiave stessa (es. `key={sistema}`) come identificatore per React, escludendo l'uso dell'indice numerico volatile.
