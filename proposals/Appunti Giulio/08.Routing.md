# 🌐 Guida Completa al Routing e alle Single Page Application (SPA) in React

Nel web tradizionale, la navigazione tra le pagine si basa su un'architettura **MPA (Multi-Page Application)**: ogni volta che clicchi su un link, il browser richiede un nuovo file HTML al server, distruggendo lo stato della pagina precedente e ricaricando interamente l'interfaccia.

React permette di scardinare questo paradigma creando le **SPA (Single Page Application)**. In una SPA, il server distribuisce un **unico file HTML vuoto** e un unico pacchetto JavaScript al primo caricamento. Quando l'utente cambia pagina, JavaScript intercetta l'azione, "smonta" i vecchi componenti dallo schermo e "monta" quelli nuovi all'istante, senza mai inviare una nuova richiesta di caricamento della pagina al server.

Per organizzare al meglio queste schermate in un progetto complesso, la convenzione prevede la creazione di una cartella dedicata: `src/pages/`. All'interno inseriamo i componenti che rappresentano intere viste dell'applicazione (es. `Index.jsx` per la Home, `Contact.jsx` per i contatti).

---

## 🛠️ 1. Il Routing Condizionale (Routing Manuale senza Librerie)

Prima di installare strumenti esterni, è fondamentale capire che il routing non è altro che un **rendering condizionale basato su uno stato**. Possiamo simulare un router primitivo usando solo `useState` e l'operatore logico `&&`.

```jsx
import { useState } from "react";
import Index from "./pages/Index";      // La nostra vista Home
import Contact from "./pages/Contact";  // La nostra vista Contatti

function App() {
  // 1. Lo stato memorizza una stringa che identifica la pagina corrente
  const [page, setPage] = useState('home');

  return (
    <>
      <nav>
        <ul>
          {/* Al click aggiorniamo lo stato, forzando il re-render di App */}
          <li onClick={() => setPage('home')}>Home</li>
          <li onClick={() => setPage('contact')}>Contatti</li>
        </ul>
      </nav>

      <main>
        {/* 2. Il Motore del Routing: Operatore Logico && */}
        {page === 'home' && <Index />}
        {page === 'contact' && <Contact />}
      </main>
    </>
  );
}

export default App;

```

### 🔍 Analisi Tecnica dell'operatore `&&`

L'operatore `&&` (Logical AND) funziona come un interruttore di sicurezza:

* React valuta l'espressione a sinistra (`page === 'home'`).
* Se l'espressione è **falsa**, React blocca immediatamente l'esecuzione del codice successivo, ignorando il componente e non renderizzando nulla sullo schermo.
* Se l'espressione è **vera**, React procede a destra ed esegue il montaggio del componente JSX (`<Index/>`).

### ⚠️ I Limiti del Routing Manuale

Questo sistema ha due enormi falle architetturali:

1. **L'URL della barra degli indirizzi resta fisso:** Se l'utente si trova su "Contatti" e ricarica la pagina (`F5`), lo stato in memoria si azzera e viene riportato alla Home.
2. **Impossibilità di Deep Linking:** Non puoi copiare l'URL di una sezione specifica per inviarlo a un amico, poiché l'indirizzo del browser non riflette la navigazione interna.

## 📦 2. Introduzione a React Router (Configurazione Dichiarativa)

Per superare i limiti del routing manuale, installiamo la libreria standard di settore tramite il terminale:

```bash
pnpm add react-router-dom

```

Passiamo così a una **Configurazione Dichiarativa**: non scriviamo più algoritmi per montare e smontare i componenti, ma dichiariamo una mappa delle rotte che la libreria dovrà gestire autonomamente, sincronizzandola con la cronologia del browser.

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import Index from "./pages/Index";
import Contact from "./pages/Contact";

function App() {
  return (
    // 1. Il Contenitore Supremo: Sincronizza l'UI con l'URL del browser
    <BrowserRouter>
      
      {/* 2. Il Vigile Urbano: Seleziona solo la prima rotta che combacia con l'URL */}
      <Routes>
        
        {/* 3. La Mappatura: path (indirizzo) ➡️ element (componente JSX) */}
        <Route path="/" element={<Index />} />
        <Route path="/contact" element={<Contact />} />

      </Routes>

    </BrowserRouter>
  );
}

export default App;

```

### 🚫 Evitare il Refresh: Dal tag `<a>` al componente `<Link>`

Se all'interno di un'app basata su React Router usi il tag HTML nativo `<a href="/contact">`, distruggerai i vantaggi della SPA. Il tag `<a>` forza il browser a richiedere una nuova pagina al server.

Per navigare correttamente, devi importare e utilizzare il componente `<Link>`:

```jsx
import { Link } from "react-router-dom";

function Navbar() {
  return (
    <nav>
      <ul>
        {/* Sostituiamo 'href' con l'attributo 'to' */}
        <li><Link to="/">Home</Link></li>
        <li><Link to="/contact">Contatti</Link></li>
      </ul>
    </nav>
  );
}

```

> ✨ **Cosa fa di speciale?** `<Link>` intercetta il click dell'utente, blocca il comportamento nativo del browser (`event.preventDefault()`) e dice a React Router di cambiare l'URL e aggiornare i componenti a schermo in modo istantaneo.

## 🏗️ 3. Layout Condivisi e l'uso di `<Outlet/>`

Nelle applicazioni moderne, alcune parti della pagina (come la barra di navigazione o il footer) rimangono identiche in ogni schermata. Ricreare questi componenti all'interno di ogni singola pagina è inefficiente. La soluzione è l'uso dei **Layout** combinati con il componente **`<Outlet/>`**.

### 🛠️ Creazione del guscio strutturale: `PrimoLayout.jsx`

```jsx
// src/layouts/PrimoLayout.jsx
import { Outlet } from 'react-router-dom';

function PrimoLayout() {
  return (
    <div className="layout-grid">
      {/* Elemento statico globale */}
      <header>
        <h1>Header Globale dello Sviluppatore</h1>
      </header>

      {/* Segnaposto Dinamico: Qui verranno inserite le pagine figlie */}
      <main>
        <Outlet />
      </main>

      {/* Elemento statico globale */}
      <footer>
        <p>Footer Globale © 2026</p>
      </footer>
    </div>
  );
}

export default PrimoLayout;

```

### 🔗 Collegamento nel Router tramite Rotte Senza Path

In `App.jsx`, puoi avvolgere le tue rotte standard dentro la rotta Layout. Omettendo l'attributo `path` dalla rotta padre, applicherai la struttura grafica a tutte le rotte figlie senza sporcare o aggiungere prefissi ai loro URL:

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";
import PrimoLayout from "./layouts/PrimoLayout";
import Index from "./pages/Index";
import Contact from "./pages/Contact";
import NotFound from "./pages/NotFound";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        
        {/* Rotta Layout: non ha path, fa solo da guscio strutturale */}
        <Route element={<PrimoLayout />}>
          {/* I componenti di queste rotte verranno iniettati dentro l'Outlet */}
          <Route path="/" element={<Index />} />
          <Route path="/contact" element={<Contact />} />
        </Route>

        {/* Gestione 404: lasciata fuori dal layout per avere una pagina pulita */}
        <Route path="*" element={<NotFound />} />

      </Routes>
    </BrowserRouter>
  );
}

```

> ⚠️ **Nota di Sviluppo (`element` vs `Component`):** Preferisci sempre la sintassi `element={<Layout/>}` rispetto a `Component={Layout}`. L'approccio `element` ti permette di passare proprietà personalizzate direttamente al componente (es. `element={<PrimoLayout tema="dark"/>}`), cosa impossibile con l'attributo `Component`.

## 📂 4. Rotte Annidate (Nested Routes) e l'attributo `index`

Le rotte annidate permettono di mappare percorsi strutturati gerarchicamente all'interno dell'URL (es. `/percorso/storia`, `/percorso/team`).

Quando creiamo un blocco di rotte figlie, usiamo l'attributo **`index`** per definire quale componente mostrare di default quando l'utente digita esattamente il percorso del padre.

```jsx
<Route path="/percorso">
  {/* URL di atterraggio: /percorso - Mostra di default <SuDiNoi /> */}
  <Route index element={<SuDiNoi />} />
  
  {/* URL specifico: /percorso/storia - Mostra <Storia /> */}
  <Route path="storia" element={<Storia />} />
</Route>

```

## 🆔 5. Parametri di Rotta Dinamici (`useParams`)

Se la tua applicazione deve mostrare i dati di 1000 utenti diversi, non puoi creare 1000 rotte manuali. Si utilizzano i **Dynamic Segments** contraddistinti dal simbolo due punti (`:`).

### 🛠️ Configurazione della Rotta Dinamica

```jsx
// Il frammento ":userId" indica che quella sezione dell'URL è una variabile
<Route path="users/:userId" element={<User />} />

```

### 🎣 Recupero e Validazione Dati: `User.jsx`

Quando un utente naviga su `/users/3`, React Router cattura il valore e lo rende disponibile tramite l'hook `useParams`. **Attenzione:** I parametri estratti dall'URL sono **sempre stringhe**. Se i tuoi ID nel database sono numerici, devi effettuare una conversione di tipo esplicita usando `Number()`.

```jsx
import { useParams } from "react-router-dom";

// Array di dati simulato
const usersData = [
  { id: 1, name: 'Giangiacomo' },
  { id: 3, name: 'Mario' }
];

function User() {
  // Estraiamo la proprietà userId (il nome deve coincidere con il path della Route)
  const { userId } = useParams();

  // Ricerca dell'utente convertendo la stringa URL in un tipo Numero
  const userFound = usersData.find(user => user.id === Number(userId));

  // 🕵️‍♂️ Gestione dell'errore se l'ID digitato non esiste
  if (!userFound) {
    return <h1>🕵️‍♂️ Nessun utente trovato</h1>;
  }

  // Schermata di successo
  return (
    <div className="profile-card">
      <h1>✨ Ciao Utente {userFound.name}</h1>
      <p>ID Profilo verificato: {userId}</p>
    </div>
  );
}

export default User;

```

## 🛑 6. Gestione degli Errori "Not Found" (404)

Per intercettare gli URL errati inseriti manualmente dagli utenti, si utilizza il percorso jolly contrassegnato dall'asterisco (`*`). Questa rotta cattura qualsiasi indirizzo che non ha trovato corrispondenza nelle definizioni precedenti.

```jsx
<Routes>
  <Route path="/" element={<Index />} />
  <Route path="/contact" element={<Contact />} />

  {/* ⚠️ Deve essere posizionata SEMPRE come ULTIMA scelta del blocco Routes */}
  <Route path="*" element={<NotFound />} />
</Routes>

```

### 🛠️ Il componente di cortesia: `NotFound.jsx`

```jsx
import { Link } from "react-router-dom";

function NotFound() {
  return (
    <div style={{ textAlign: 'center', marginTop: '100px' }}>
      <h1>404 - Pagina non trovata 🕵️‍♂️</h1>
      <p>La pagina che stai cercando è inesistente o è stata spostata.</p>
      {/* Sfruttiamo il Link per rientrare nella SPA senza refresh del sistema */}
      <Link to="/">Torna alla Home dell'applicazione</Link>
    </div>
  );
}

export default NotFound;

```

## 📓 Schema dei Concetti Chiave per gli Appunti

### 🧭 Strumenti di Navigazione

* **`BrowserRouter`**: Il componente radice che si mette a capo dell'intera applicazione per abilitare la lettura dell'URL.
* **`Routes`**: Il blocco decisionale che esamina l'indirizzo attuale e seleziona una sola rotta da mostrare.
* **`Route`**: L'elemento di mappatura che collega una stringa di testo (`path`) al relativo componente visivo (`element`).
* **`<Link to="...">`**: Sostituto del tag `<a>`. Esegue la navigazione interna manipolando l'URL senza mai causare il refresh della pagina.

---

### 📂 Funzionalità Avanzate

* **`<Outlet/>`**: Il punto di iniezione posizionato all'interno di un componente Layout in cui verranno renderizzate le sotto-rotte.
* **`index`**: Attributo che imposta un componente figlio come schermata predefinita per l'URL del padre.
* **`path="*"`**: Il carattere jolly da posizionare in fondo alle rotte per catturare gli indirizzi inesistenti e prevenire schermate bianche d'errore.
* **`useParams()`**: Hook per estrarre le variabili dinamiche passate all'interno dell'URL (restituisce sempre stringhe).
