
## 🧬 2.3 Operatori logici

### 📝 2.3.1 Gli Operatori Logici

Tuttavia, nei programmi reali, spesso una decisione non dipende da un solo fattore, ma da più condizioni contemporaneamente. Per unire, combinare o invertire queste domande logiche, utilizziamo gli operatori logici.
Gli operatori logici prendono i risultati true o false dei singoli confronti e li legano insieme seguendo le regole della logica booleana.

In JavaScript ne utilizziamo principalmente tre:

- **AND (&&):** Funziona come un "E contemporaneamente". Tutta la condizione è vera solo se tutti i singoli pezzi sono veri.
    
- **OR (||):** Funziona come un "Oppure". Tutta la condizione è vera se almeno uno dei pezzi è vero. Si scrive usando due barre verticali (chiamate pipe).
    
- **NOT (!):** Funziona come un "Inverti/Negazione". Si mette davanti a un valore booleano per ribaltarlo (trasforma true in false e viceversa).
    

Grazie agli operatori logici possiamo evitare di scrivere troppi if uno dentro l'altro, riducendo le righe di codice e rendendo il programma molto più pulito e facile da leggere.

### 💻 2.3.2 Esempio base

Vediamo come funzionano l'operatore AND (&&) e l'operatore OR (||) quando combinano variabili booleane.

```JavaScript
let haBiglietto = true;
let haDocumento = false;

// Per entrare servono ENTRAMBI i requisiti (AND)
let puoEntrareAlConcerto = haBiglietto === true && haDocumento === true;

// Per ricevere lo sconto basta avere il codice coupon OPPURE essere un nuovo utente (OR)
let haCoupon = true;
let eNuovoUtente = false;
let ottieneSconto = haCoupon === true || eNuovoUtente === true;

console.log(puoEntrareAlConcerto); // Stamperebbe false
console.log(ottieneSconto);       // Stamperebbe true
```

🔍 **Spiegazione dell'esempio:**

- `puoEntrareAlConcerto`: L'operatore && nota che `haBiglietto` è vero, ma vede che `haDocumento` è falso. Poiché l'AND esige che tutte le condizioni siano vere, il controllo totale fallisce e restituisce `false`.
    
- `ottieneSconto`: L'operatore || vede che `haCoupon` è vero. Anche se `eNuovoUtente` è falso, all'OR basta una sola verità per attivarsi. Di conseguenza, restituisce `true`.
    

### 🛠️ 2.3.3 Esempio pratico

Immaginiamo il sistema di sblocco di una piattaforma di streaming video. Un utente può riprodurre un film premium solo se possiede l'abbonamento mensile E se il suo dispositivo è connesso a internet. In aggiunta, se l'utente è un amministratore del sito, deve poter vedere il video in ogni caso (anche se i controlli precedenti falliscono).

```JavaScript
// Stato dell'utente attuale
let haAbbonamento = true;
let isConnessoAInternet = false;
let isAmministratore = true;

// Condizione combinata con uso delle parentesi per la precedenza
if ((haAbbonamento === true && isConnessoAInternet === true) || isAmministratore === true) {
  console.log("Riproduzione video avviata con successo!");
} else {
  console.log("Impossibile avviare il video. Controlla la connessione o il tuo abbonamento.");
}
```

🔍 **Spiegazione dell'esempio:**

- JavaScript legge prima l'espressione dentro le parentesi tonde più interne: `haAbbonamento === true && isConnessoAInternet === true`. Diventa `true && false`, che genera come risultato parziale `false`.
    
- Successivamente, il programma analizza la seconda parte della riga con l'operatore OR: `(Risultato Precedente) || isAmministratore === true`. Diventa quindi `false || true`.
    
- Visto che l'operatore è un `||` (OR) e la variabile `isAmministratore` è vera, l'intera riga si trasforma in un definitivo `true`.
    
- Il computer entra nella stanza dell'if e stampa: `Riproduzione video avviata con successo!`.
    

### ⚠️ 2.3.4 Errori comuni

- **Scrivere un solo carattere (& o |):** Questo è un errore di distrazione molto frequente. In JavaScript, scrivere un solo simbolo cambia completamente il tipo di operazione (diventa un calcolo matematico a livello di bit detto Bitwise), rompendo la logica dell'if. Ricordati di scriverli sempre doppi: `&&` e `||`.
    
- **Abusare dell'operatore NOT (!) incrociando i pensieri:** L'operatore `!` ribalta la logica. Scrivere `if (!isNonDisponibile)` crea una doppia negazione che confonde l'occhio umano in fase di lettura. È sempre meglio dare alle variabili nomi positivi (es: `isDisponibile`) per evitare di intrecciare i ragionamenti logici.
    
- **Dimenticare le parentesi per stabilire la precedenza:** Proprio come nelle espressioni matematiche dove le moltiplicazioni si calcolano prima delle somme, nel codice l'operatore `&&` ha la precedenza su `||`. Se vuoi che un gruppo di controlli in OR venga calcolato prima, devi isolarlo usando le parentesi tonde `( )`.
    

## 📌 2.4 Approfondimento leggero: Il cortocircuito logico (Short-Circuit Evaluation)

JavaScript è un linguaggio ottimizzato per risparmiare tempo ed energia. Quando valuta gli operatori logici, usa un trucco chiamato cortocircuito:

- Con l'operatore **&&**, se JavaScript incontra la prima condizione ed è falsa, si ferma subito. Sa già che l'intera riga non potrà mai essere vera, quindi non perde tempo a leggere le condizioni successive.
    
- Con l'operatore **||**, se incontra la prima condizione ed è vera, si ferma immediatamente. Sa già che l'intera riga sarà vera (visto che all'OR basta una sola verità), saltando i controlli successivi.

🚀 **Key Takeaways del Giorno**

- **AND (&&) esige la perfezione:** Restituisce `true` solo ed esclusivamente se tutte le singole condizioni separate dal simbolo sono contemporaneamente vere.
    
- **OR (||) cerca una via d'uscita:** Restituisce `true` non appena trova almeno una condizione vera all'interno della lista, ignorando le parti false.
    
- **NOT (!) ribalta la realtà:** Si posiziona davanti a un valore o a una variabile booleana per trasformare il vero in falso e il falso in vero.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                     | **"Spiega Brutta"**                                                                     |
| ------------------------- | --------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **Operatore Logico**      | Simbolo utilizzato per combinare o modificare più espressioni booleane.     | Lo strumento per unire più domande insieme e creare regole di controllo più articolate. |
| **&& (AND)**              | Operatore logico di congiunzione; richiede la verità di tutti gli operandi. | Il controllore severo: o tutti i requisiti sono validi, o la risposta finale è no.      |
| **\| (OR)**               | Operatore logico di disgiunzione; richiede la verità di almeno un operando. | Il controllore flessibile: basta che una sola cosa vada bene per dare il via libera.    |
| **! (NOT)**               | Operatore logico di negazione unaria; inverte il valore booleano corrente.  | Il bastian contrario del codice: se una cosa è accesa, lui la spegne.                   |
