## 📝 1. **Definizione di Condizione**

Una condizione in JavaScript è un’espressione il cui risultato è un dato di tipo `boolean`: `true` o `false`. Il suo compito è permettere al programma di scegliere un comportamento diverso in base ai dati disponibili.

In altre parole, una condizione è come una domanda che il programma si pone in tempo reale:

- "Questo valore è maggiore di 10?"
    
- "L’utente ha inserito i dati corretti?"
    
- "Ha diritto ad accedere a questa sezione?"
    

👉 **Il risultato di una condizione è sempre uno tra due valori booleani:**

- **true (vero):** La condizione è soddisfatta, quindi il codice specifico può essere eseguito.
    
- **false (falso):** La condizione non è soddisfatta, quindi il codice viene saltato (o il programma cambia comportamento).
    

Le condizioni sono fondamentali perché rendono il programma dinamico, cioè capace di reagire a situazioni diverse. Senza di esse, il programma eseguirebbe sempre le stesse istruzioni senza mai adattarsi all’utente o ai dati.

## 💻 2. **Esempio base**

Vediamo la struttura fondamentale per gestire una decisione: il costrutto `if` (se).

```js
let eta = 18;

if (eta >= 18) {
  console.log("Sei maggiorenne");
}
```

🔍 **Spiegazione dell'esempio:**

- `eta >= 18` è la nostra condizione inserita tra le parentesi tonde.
    
- JavaScript confronta il valore della variabile `eta` con il numero 18.
    
- Se il risultato è `true`, il blocco di codice racchiuso dentro le parentesi graffe `{ }` viene eseguito.
    
- Se il risultato è `false`, il blocco viene ignorato completamente.
    
- In questo caso specifico: se `eta` è 18 o più viene mostrato il messaggio; se `eta` è minore di 18 non succede assolutamente nulla.
    
## 🔄 3. **Esempio con else**

Cosa succede se vogliamo gestire anche lo scenario alternativo? In questo caso aggiungiamo la parola chiave `else` (altrimenti) per creare una seconda stanza logica.


```js
let eta = 15;

if (eta >= 18) {
  console.log("Sei maggiorenne");
} else {
  console.log("Sei minorenne");
}
```

🔍 **Spiegazione dell'esempio:**

- Il programma prima valuta la condizione `eta >= 18`.
    
- Se è vera (`true`), esegue il primo blocco di codice.
    
- Se è falsa (`false`), salta la prima stanza e va direttamente ad eseguire il blocco `else`.
    
- Questo meccanismo permette di gestire due scenari opposti in modo pulito, senza lasciare casi “vuoti”.
    

## ⚠️ 4. **Errori comuni**

- **Usare un solo simbolo di uguale = dentro l'if:** Questo è l'errore più diffuso. Scrivere `if (utenteLoggato = true)` non esegue un controllo, ma cambia il valore della variabile forzandola a true, distruggendo la logica della condizione. Per confrontare usa sempre i tre uguali `===` o l'operatore relazionale corretto. 
    
- **Mettere il punto e virgola subito dopo le parentesi tonde:** Scrivere `if (eta >= 18); { ... }` interrompe bruscamente il controllo. JavaScript penserà che l'istruzione finisca su quel punto e virgola ed eseguirà sempre il codice nelle parentesi graffe, qualunque sia il valore della variabile.
    
- **Dimenticare le parentesi graffe { }:** Se non metti le parentesi graffe dopo l'if, JavaScript eseguirà in modo condizionato solo la primissima riga subito sotto, mentre le righe successive verranno eseguite sempre, creando bug imprevedibili.
    

## 💡 5. **Approfondimento leggero: Dove si usano le condizioni nel mondo reale**

Le condizioni sono una delle basi della programmazione e si usano ogni volta che il programma deve prendere una decisione automatica per adattarsi all'utente.

Esempi tipici nel software moderno:

- Verificare se un utente è loggato prima di farlo accedere a una pagina privata.
    
- Controllare se un campo di un modulo di registrazione è stato compilato correttamente.
    
- Decidere se mostrare un messaggio di errore rosso o una notifica verde di successo.
    
- Confrontare valori dinamici (numeri, testi, stati dell'applicazione).

## 7. **Key Takeaways del Giorno**

- **Le condizioni fanno decidere il codice:** Permettono al programma di non essere una lista fissa di comandi, ma di reagire e adattarsi a situazioni diverse.
    
- **Sempre vero o falso:** Ogni condizione produce rigorosamente un risultato booleano (`true` o `false`).
    
- **if/else gestisce i casi:** Consente di coprire più scenari opposti nello stesso flusso, garantendo che il programma sappia sempre cosa fare.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                       | **"Spiega Brutta"**                                                        |
| ------------------------- | ------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **Condizione**            | Espressione valutata dal programma come vera o falsa.         | Una domanda che JavaScript si fa in tempo reale per capire cosa fare dopo. |
| **if / else**             | Struttura di controllo del flusso del programma.              | Il bivio: “se succede A fai questo, altrimenti fai quest'altro”.           |
| **Boolean**               | Tipo di dato con solo due valori possibili: `true` o `false`. | L'interruttore acceso/spento del codice.                                   |
| **Blocco di codice**      | Gruppo di istruzioni racchiuse tra parentesi graffe `{ }`.    | Il pezzo di programma chiuso in una stanza che parte solo quando serve.    |
