## 🛠️ 7.2 Sintassi degli Oggetti (Object Literal)

Il modo più comune, semplice e utilizzato per creare un oggetto in JavaScript si chiama **Object Literal** (Oggetto Letterale). Consiste nel dichiarare una variabile e assegnarle direttamente un blocco di parentesi graffe `{ }`.

All'interno di queste parentesi, elenchiamo le caratteristiche dell'oggetto sotto forma di coppie _chiave: valore_, separando ogni coppia con una virgola.

### 🧬 7.2.1 Anatomia della Sintassi

Analizziamo le regole di scrittura fondamentali:

- **La Chiave (o Proprietà):** È il nome del dato (l'identificatore). Di solito si scrive come testo semplice senza virgolette. Se però il nome della chiave è composto da due parole separate da uno spazio o da caratteri speciali, allora va tassativamente racchiuso tra virgolette (es. `"colore carrozzeria"`).
    
- **I Due Punti (`:`):** Separano nettamente il nome della chiave dal suo rispettivo valore.
    
- **Il Valore:** È il dato vero e proprio. Un oggetto è incredibilmente flessibile: come valore può ospitare qualsiasi tipo di dato esistente in JavaScript (stringhe, numeri, booleani, array e persino altri oggetti annidati).
    

```JavaScript
// Creazione di un oggetto con proprietà complesse e annidate
const miaAuto = {
  marca: "Fiat",
  modello: "500",
  anno: 2020,
  "colore carrozzeria": "Rosso", // Chiave con spazio: richiede le virgolette
  inFunzione: true,
  accessori: ["Climatizzatore", "Sensori di parcheggio"], // Un array come valore
  
  // OGGETTO ANNIDATO: il valore di questa chiave è un altro oggetto
  motore: {
    cilindrata: 1200,
    alimentazione: "Ibrida",
    cavalli: 69, // <- Trailing Comma: la virgola finale qui è lecita e raccomandata!
  }, 
};
```

> 💡 **Nota di stile (Trailing Comma):** Inserire la virgola anche nell'ultima proprietà prima di chiudere la parentesi graffa è una pratica consigliata nello sviluppo moderno. Evita errori di sintassi futuri se deciderai di aggiungere nuove righe in fondo e rende i messaggi di commit su Git molto più puliti.

### ⚙️ 7.2.2 Differenza tra Proprietà e Metodi

I dati che inseriamo dentro un oggetto si dividono in due grandi categorie logiche:

- **Proprietà:** Quando il valore associato alla chiave è un dato passivo (una stringa, un numero, un array, un booleano). Rappresentano le caratteristiche statiche dell'oggetto (es. il colore dell'auto).
    
- **Metodi:** Quando il valore associato alla chiave è una funzione. Rappresentano le azioni che l'oggetto può compiere (es. accendere il motore).
    

```JavaScript
const smartphone = {
  // Queste sono PROPRIETÀ (Dati statici)
  marca: "Apple",
  modello: "iPhone",
  
  // Questo è un METODO (Un'azione / Funzione)
  accendi: function() {
    console.log("Lo schermo si illumina... Smartphone acceso!");
  }
};
```

### ⚠️ 7.2.3 Errori comuni

- **Usare il punto e virgola al posto della virgola:** Dentro un oggetto, le coppie si separano sempre con la virgola `,`. Scrivere `const utente = { nome: "Mario"; eta: 30 };` (con il punto e virgola dentro le graffe) bloccherà il programma lanciando un errore di sintassi (`SyntaxError`).
    
- **Dimenticare le virgolette per chiavi con caratteri speciali:** Se provi a scrivere una chiave con uno spazio o un trattino senza usare le virgolette, come `anno-acquisto: 2024`, JavaScript si confonderà pensando che tu stia provando a fare una sottrazione logica, mandando in blocco il codice.

🚀 **Key Takeaways del Giorno**

- **Le graffe definiscono l'oggetto:** Ricorda che `[]` crea un array, mentre `{}` crea un oggetto.
    
- **Separa con la virgola:** Ogni singola caratteristica (coppia) dell'oggetto deve finire con una virgola per far capire a JavaScript dove inizia la successiva.
    
- **I metodi sono azioni:** Se una chiave ha come valore una funzione, quella chiave diventa un metodo e serve a far "fare qualcosa" al tuo oggetto.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                                      | **"Spiega Brutta"**                                                                                                    |
| ------------------------- | ------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------------------- |
| **Object Literal**        | Sintassi di dichiarazione diretta di un oggetto mediante l'uso di parentesi graffe e un elenco di proprietà. | Scrivere un oggetto a mano nel codice aprendo e chiudendo le parentesi graffe.                                         |
| **Proprietà (Property)**  | Associazione tra un nome (chiave) e un valore memorizzato all'interno di un oggetto.                         | Una caratteristica fissa e ferma memorizzata dentro l'oggetto (es. il colore, l'età).                                  |
| **Metodo (Method)**       | Una funzione che è memorizzata come proprietà all'interno di un oggetto.                                     | Un'azione che l'oggetto sa fare. È una funzione travestita da proprietà.                                               |
| **Nested Object**         | Struttura in cui il valore di una proprietà di un oggetto è a sua volta un riferimento a un altro oggetto.   | Un oggetto a scatole cinesi: un oggetto infilato dentro un altro oggetto per descrivere dettagli ancora più specifici. |
