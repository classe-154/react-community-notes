## 🧬 7.6 Destructuring (Spacchettamento degli Oggetti)

Quando lavoriamo con oggetti molto grandi, andare a leggere continuamente le proprietà usando la Dot Notation (`oggetto.proprieta`) può rendere il codice lungo, ripetitivo e pesante da leggere.

Il **Destructuring** (Destrutturazione) è una sintassi moderna ed elegantissima che permette di "estrarre" le proprietà da un oggetto e trasformarle istantaneamente in variabili locali pronte all'uso, il tutto in un'unica e sola riga di codice.

### 🆚 7.6.1 Il Metodo Classico vs Destructuring

Immagina di avere l'oggetto di un utente e di voler salvare il suo nome e la sua età in due variabili separate.

#### Modo Classico (Lungo)

```JavaScript
const utente = {
  nome: "Federica",
  eta: 22,
  citta: "Napoli"
};

// Devo usare il punto per ogni singola variabile
const nome = utente.nome;
const eta = utente.eta;

console.log(nome); // Stampa: Federica
```

#### Modo Moderno (Con il Destructuring)

Mettiamo le parentesi graffe `{ }` a sinistra dell'uguale. In questo modo diciamo a JavaScript: _"Prendi l'oggetto a destra, apri i cassetti che ti elenco e crea delle variabili che si chiamano come le chiavi"_.

```JavaScript
const utente = {
  nome: "Federica",
  eta: 22,
  citta: "Napoli"
};

// Estraiamo 'nome' ed 'eta' in un colpo solo
const { nome, eta } = utente;

// Ora le variabili esistono da sole, senza bisogno di scrivere utente.nome!
console.log(nome); // Stampa: Federica
console.log(eta);  // Stampa: 22
```

#### Schema Visivo dello Spacchettamento

```Plaintext
OGGETTO ORIGINALE (a destra dell'uguale)
Contiene le chiavi: nome, eta, citta

      │
      ├─────► La chiave nome viene estratta e diventa ─────► Variabile nome
      │                                                      (Valore: Federica)
      │
      └─────► La chiave eta viene estratta e diventa ──────► Variabile eta
                                                             (Valore: 22)

NUOVE VARIABILI LOCALI (a sinistra dell'uguale)
Ora le variabili nome ed eta esistono da sole nel codice.
```

### 🔄 7.6.2 Rinominare le Variabili durante l'Estrazione

Cosa succede se hai già una variabile che si chiama `nome` nel tuo codice e vuoi evitare un conflitto di nomi? Il destructuring ti permette di estrarre una proprietà e rinominarla al volo.

La sintassi prevede l'uso dei due punti `:` dopo il nome originale della chiave, seguiti dal nuovo nome che vuoi dare alla variabile finale.

```JavaScript
const prodotto = {
  id: 101,
  prezzo: 15
};

// Estraiamo la chiave 'prezzo' ma la salviamo in una variabile chiamata 'costoArticolo'
const { prezzo: costoArticolo } = prodotto;

console.log(costoArticolo); // Stampa: 15
// NOTA: la variabile 'prezzo' non esiste, esiste solo 'costoArticolo'!
```

### 🛟 7.6.3 Valori di Default (Salvagente)

Se provi a estrarre una chiave che non esiste dentro l'oggetto, la tua nuova variabile sarà `undefined`. Per evitare questo problema, puoi impostare un valore di default (un valore di riserva) usando l'operatore di assegnazione `=` durante lo spacchettamento.

```JavaScript
const profilo = {
  nomeUtente: "CyberPunk",
  livello: 5
};

// Se la chiave 'ruolo' non esiste nell'oggetto, assegna il valore "Ospite" di default
const { nomeUtente, ruolo = "Ospite" } = profilo;

console.log(ruolo); // Stampa: Ospite (perché non era presente nell'oggetto)
```

### ⚠️ 7.6.4 Errori comuni

- **Confondere la creazione di un oggetto con il destructuring:** Le parentesi graffe `{ }` ingannano l'occhio del programmatore. Se le graffe si trovano a destra dell'uguale stanno _creando_ un oggetto literal; se si trovano a sinistra dell'uguale stanno _distruggendo_ (spacchettando) un oggetto esistente.
    
- **Provare a fare il destructuring di un oggetto inesistente (`null` o `undefined`):** Se provi a scrivere `const { nome } = datiUtente;` ma la variabile `datiUtente` è vuota o non è ancora stata definita, il browser andrà in crash totale restituendo un errore bloccante di tipo (`TypeError: Cannot destructure property... of undefined`).

🚀 **Key Takeaways del Giorno**

- **Le graffe a sinistra estraggono:** Usa la destrutturazione per eliminare la ripetizione ossessiva del punto `.` quando devi estrarre e usare molti dati da un singolo oggetto.
    
- **I nomi devono coincidere:** Di base, il nome della variabile inserita dentro le graffe deve essere scritto esattamente come la chiave presente nell'oggetto di origine.
    
- **Usa i due punti per rinominare:** Se vuoi cambiare nome alla variabile locale finale per evitare conflitti, usa la sintassi `chiaveOriginale: nuovoNome`.
    

📖 **Glossario**

| **Termine Istituzionale**    | **Definizione Formale**                                                                                                        | **"Spiega Brutta"**                                                                                                             |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| **Destructuring Assignment** | Espressione JavaScript che consente di estrarre dati da array o oggetti in variabili distinte mediante una sintassi speculare. | Spacchettare una scatola: tiri fuori i singoli dati da un oggetto e li trasformi subito in variabili separate e pronte all'uso. |
| **Valore di Default**        | Valore preimpostato che viene assegnato automaticamente a una variabile qualora il dato originale risulti assente.             | Una ruota di scorta: il valore che usi per sicurezza se nel cassetto dell'oggetto non c'è scritto nulla.                        |
| **Alias / Ridenominazione**  | Meccanismo sintattico per assegnare un nome differente a una risorsa o variabile durante la sua fase di binding.               | Cambiare il nome del dato mentre lo tiri fuori dall'oggetto per non farlo confondere con altre variabili già esistenti.         |
