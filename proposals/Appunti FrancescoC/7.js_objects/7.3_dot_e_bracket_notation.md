## 🔍 7.3 Leggere e Modificare i Dati (Dot e Bracket Notation)

Una volta creato un oggetto, abbiamo bisogno di accedere alle sue proprietà o ai suoi metodi per utilizzarli nel programma. JavaScript ci mette a disposizione due strumenti per farlo: la **Dot Notation** (la notazione con il punto) e la **Bracket Notation** (la notazione con le parentesi quadre).

### 🟢 7.3.1 La Dot Notation (La notazione con il punto)

La Dot Notation è il modo più semplice, elegante e comune per accedere ai dati. Si scrive il nome dell'oggetto, si mette un punto `.` e si scrive direttamente il nome della chiave che si vuole leggere o modificare.

```JavaScript
const studente = {
  nome: "Valerio",
  materia: "JavaScript",
  voto: 30
};

// 1. Lettura dei dati
console.log(studente.nome); // Stampa: Valerio

// 2. Modifica di un dato esistente
studente.voto = 31; 

// 3. Aggiunta di una nuova proprietà da zero
studente.promosso = true;

// 4. Eliminazione definitiva di una proprietà (Chiave e Valore) 
delete studente.materia; 
console.log(studente.materia); // Stampa: undefined (La chiave non esiste più!)
```

### 🟡 7.3.2 La Bracket Notation (La notazione con le quadre)

La Bracket Notation fa la stessa identica cosa del punto, ma usa le parentesi quadre `[ ]`. All'interno delle quadre dobbiamo passare il nome della chiave sotto forma di stringa (racchiusa tra virgolette).

```JavaScript
const docente = {
  nome: "Alessandro",
  corso: "Web Design"
};

// Lettura tramite stringa nelle quadre
console.log(docente["corso"]); // Stampa: Web Design
```

### 🚨 7.3.3 Quando le parentesi quadre sono OBBLIGATORIE?

Se fanno la stessa cosa, perché esistono entrambe? Esistono due situazioni ingegneristiche specifiche in cui la Dot Notation fallisce e siamo obbligati a usare le parentesi quadre:

#### Caso 1: Le chiavi con spazi o caratteri speciali

Se una chiave è stata registrata con uno spazio vuoto (es. `"anno acquisto"`), scrivere `oggetto.anno acquisto` provocherà un errore di sintassi. Il punto non può gestire gli spazi.

```JavaScript
const smartphone = {
  marca: "Samsung",
  "anno acquisto": 2024
};

// ERRORE: console.log(smartphone.anno acquisto);

// CORRETTO:
console.log(smartphone["anno acquisto"]); // Stampa: 2024
```

#### Caso 2: Usare una variabile per cercare la chiave dinamicamente

Se il nome della chiave che devi cercare non lo conosci a priori, ma si trova memorizzato dentro un'altra variabile, il punto non funziona. Se usi il punto, JavaScript cercherà una chiave che si chiama letteralmente come il nome della variabile stessa.

```JavaScript
const utente = {
  nome: "Chiara",
  Stato: "Italia"
};

const chiaveDaCercare = "nome";

// Usando il punto: cerca una chiave che si chiama proprio "chiaveDaCercare" (e non esiste)
console.log(utente.chiaveDaCercare); // Stampa: undefined

// Usando le quadre senza virgolette: legge il VALORE della variabile ("nome") e cerca quella chiave
console.log(utente[chiaveDaCercare]); // Stampa: Chiara
```

### 🔊 7.3.4 Come invocare i Metodi

Se la chiave a cui accedi contiene una funzione (un metodo), per farla eseguire devi ricordarti di aggiungere sempre le parentesi tonde `()` alla fine del comando, sia che usi il punto sia che usi le quadre.

```JavaScript
const cane = {
  nome: "Rex",
  abbaia: function() {
    console.log("Bau Bau!");
  }
};

// Invocazione classica con il punto
cane.abbaia(); // Stampa: Bau Bau!

// Invocazione con le quadre
cane["abbaia"](); // Stampa: Bau Bau!
```

### ⚠️ 7.3.5 Errori comuni

- **Dimenticare le virgolette dentro le quadre con chiavi fisse:** Scrivere `studente[nome]` senza virgolette significa dire a JavaScript di cercare una variabile chiamata `nome`. Se quella variabile non esiste, il programma andrà in blocco totale con un errore di riferimento (`ReferenceError`). Se cerchi la chiave fissa, scrivi sempre `studente["nome"]`.
    
- **Cercare di usare il punto con chiavi numeriche:** Se un oggetto ha delle chiavi che sono numeri (cosa rara ma possibile), scrivere `oggetto.1` è un errore di sintassi. In questo scenario devi usare obbligatoriamente la sintassi `oggetto[1]`.

🚀 **Key Takeaways del Giorno**

- **Usa il punto per la normalità:** Se le chiavi sono scritte in modo classico (senza spazi), usa sempre la Dot Notation per la massima leggibilità e pulizia del codice.
    
- **Usa le quadre per la dinamicità:** Se il nome della chiave dipende dal valore di una variabile o contiene spazi o caratteri speciali, le parentesi quadre sono la tua unica ancora di salvataggio.
    
- **Le tonde attivano i metodi:** Accedere a un metodo senza mettere le parentesi tonde `()` non eseguirà l'azione, ma ti restituirà semplicemente la definizione testuale della funzione stessa.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                                                                        | **"Spiega Brutta"**                                                                                                |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **Dot Notation**          | Sintassi di accesso alle proprietà di un oggetto che utilizza l'operatore punto seguito dal nome letterale della proprietà.                    | Andare a prendere un dato dentro l'oggetto usando il punto (es. `auto.colore`).                                    |
| **Bracket Notation**      | Sintassi di accesso alle proprietà che utilizza le parentesi quadre contenenti una stringa o una variabile che valuta il nome della proprietà. | Andare a prendere un dato usando le quadre e il nome della chiave come testo o variabile (es. `auto["colore"]`).   |
| **Property Accessor**     | Operatori nativi di JavaScript che consentono di accedere alle proprietà o ai metodi memorizzati in un oggetto.                                | I modi ufficiali (punto o quadre) che il linguaggio ti offre per aprire i cassetti di un oggetto.                  |
| **Delete Operator**       | Operatore JavaScript che rimuove una proprietà da un oggetto, liberando la memoria associata.                                                  | Il tasto cancella per gli oggetti: elimina completamente sia l'etichetta (chiave) che il suo contenuto dal record. |
