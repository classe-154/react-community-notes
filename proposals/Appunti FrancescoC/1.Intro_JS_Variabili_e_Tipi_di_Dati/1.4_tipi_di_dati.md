## 🔣 1.4 Tipi di dati

### 🧠 1.4.1 Cosa sono i tipi di dati e a cosa servono

I tipi di dati sono le diverse categorie di informazioni che JavaScript può comprendere e memorizzare. Dire a JavaScript il tipo di dato che stiamo usando serve a fargli capire quali operazioni può fare con quella scatola.

Ad esempio:

- Se JavaScript sa che in due scatole ci sono dei numeri, userà il simbolo `+` per fare una somma matematica (es. 5 + 5 fa 10).
    
- Se sa che ci sono dei pezzi di testo, userà lo stesso simbolo `+` per unire le parole (es. "5" + "5" fa "55").
    

Per iniziare, i tre tipi di dati più importanti da conoscere sono:

- **Stringhe (testo):** Qualsiasi lettera, parola o frase racchiusa tra virgolette.
    
- **Numeri:** Numeri interi o con la virgola (che in programmazione si scrive con il punto `.`), scritti senza virgolette.
    
- **Booleani:** Una categoria speciale che può contenere solo due valori: `true` (vero) o `false` (falso). Servono come interruttori per prendere decisioni nel codice.
    

### 💻 1.4.2 Esempio base

Vediamo come riempire le nostre scatole con questi tre diversi tipi di dati fondamentali.



```JavaScript
// 1. Tipo di dato: Stringa (Testo)
let filmPreferito = "Ritorno al Futuro";

// 2. Tipo di dato: Numero
let annoUscita = 1985;

// 3. Tipo di dato: Booleano (Vero o Falso)
let eUnBelFilm = true;

console.log(filmPreferito);
console.log(annoUscita);
console.log(eUnBelFilm);
```

🔍 **Spiegazione dell'esempio:**

- `let filmPreferito = "Ritorno al Futuro";`: Creiamo una stringa. Le virgolette fanno capire a JavaScript che questo è testo puro e semplice, non un comando da eseguire.
    
- `let annoUscita = 1985;`: Creiamo un numero. Non mettiamo le virgolette. In questo modo JavaScript sa che con questa scatola può fare calcoli matematici (come calcolare quanti anni fa è uscito il film).
    
- `let eUnBelFilm = true;`: Creiamo un booleano. Rappresenta una verità. Non servono le virgolette perché `true` e `false` sono parole speciali che JavaScript riconosce da solo.
    

### 🛠️ 1.4.3 Esempio pratico

Immaginiamo di creare il profilo riassuntivo per un utente di un sito web di fitness. Dobbiamo salvare il suo nome, il suo peso e verificare se ha completato l'allenamento di oggi.

```JavaScript
// Dati dell'utente
let nomeUtente = "Sara"; // Stringa
let pesoKg = 62.5;       // Numero (usiamo il punto per i decimali)
let allenamentoFatto = false; // Booleano (inizialmente è falso)

// Il giorno dopo, Sara finisce l'allenamento! Cambiamo l'interruttore su vero
allenamentoFatto = true;

console.log("Nome: " + nomeUtente);
console.log("Peso attuale: " + pesoKg + " kg");
console.log("L'utente ha completato l'allenamento? " + allenamentoFatto);
```

🔍 **Spiegazione dell'esempio:**

- Il programma crea tre scatole con tre tipi di dati diversi: una stringa ("Sara"), un numero con la virgola scritto stile americano con il punto (`62.5`), e un interruttore booleano impostato su `false`.
    
- Quando l'utente completa l'attività, il programma va a modificare la scatola `allenamentoFatto` e gira l'interruttore su `true`. I booleani sono perfetti per questo: accesi o spenti, senza vie di mezzo.
    
- Alla fine, `console.log` unisce i testi alle variabili e stampa un riepilogo leggibile per noi sviluppatori.
    

### ⚠️ 1.4.4 Errori comuni

- **Mettere i numeri tra virgolette per errore:** Se scrivi `let eta = "20";`, JavaScript non lo vede come un numero, ma come testo. Se provi a fare `eta + 1`, il risultato non sarà 21, ma diventerà la scritta "201", perché ha semplicemente attaccato il numero 1 alla fine del testo.
    
- **Mettere i booleani tra virgolette:** Scrivere `let attivo = "true";` trasforma l'interruttore in una stringa di testo qualunque. Questo rompe i meccanismi automatici di controllo che JavaScript applica ai veri valori `true` o `false`.
    
- **Usare la virgola nei numeri:** In JavaScript, scrivere `let prezzo = 12,50;` provoca un errore di sintassi che blocca il programma. Ricordati di usare sempre il punto fermo per i decimali: `let prezzo = 12.50;`.
    

## 📌 1.5 Metodi dei dati

### 🔍 1.5.1 Approfondimento leggero: Il tipo "Null" e il tipo "Undefined"

Oltre ai tre tipi principali, esistono altre due scatole speciali in JavaScript che indicano la mancanza di qualcosa:

- **undefined:** Significa che la scatola è stata creata, ma è ancora completamente vuota. Non ci abbiamo messo dentro ancora nulla (es. `let x;`).
    
- **null:** Significa che abbiamo deciso intenzionalmente di svuotare la scatola o di lasciarla vuota per il momento. È una scelta intenzionale del programmatore per dire "qui dentro non c'è nessun valore".

🚀 **Key Takeaways del Giorno**

- **Le virgolette fanno il testo:** Tutto ciò che si trova tra virgolette è una stringa (testo). Se levi le virgolette, JavaScript cerca di interpretarlo come numero, booleano o nome di una variabile.
    
- **I numeri usano il punto:** Per i numeri con la virgola si usa tassativamente il punto (`.`).
    
- **I Booleani sono interruttori:** Esistono solo due valori possibili, `true` e `false`, perfetti per gestire risposte del tipo sì/no o acceso/spento.
