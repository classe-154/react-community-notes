## 🔣 1.4 Tipi di dati

### 🧠 1.4.1 Cosa sono i tipi di dati e a cosa servono

I tipi di dati sono le diverse categorie di informazioni che JavaScript può comprendere e memorizzare. Dire a JavaScript il tipo di dato che stiamo usando serve a fargli capire quali operazioni può fare con quella scatola.

Ad esempio:

- Se JavaScript sa che in due scatole ci sono dei numeri, userà il simbolo `+` per fare una somma matematica (es. `5 + 5` fa `10`).
    
- Se sa che ci sono dei pezzi di testo, userà lo stesso simbolo `+` per unire le parole (es. `"5" + "5"` fa `"55"`).
    

Per iniziare, i tre tipi di dati più importanti da conoscere sono:

- **Stringhe (testo):** Qualsiasi lettera, parola o frase racchiusa tra virgolette.
    
- **Numeri:** Numeri interi o con la virgola (che in programmazione si scrive con il punto `.`), scritti senza virgolette.
    
- **Booleani:** Una categoria speciale che può contenere solo due valori: `true` (vero) o `false` (falso). Servono come interruttori per prendere decisioni nel codice.
    

### 💻 1.4.2 Esempio base

Vediamo come riempire le nostre scatole con questi tre diversi tipi di dati fondamentali.

```JavaScript
// 1. Tipo di dato: Stringa (Testo)
let filmPreferito = "Ritorno al Futuro";

// 2. Tipo di dato: Numero
let annoUscita = 1985;

// 3. Tipo di dato: Booleano (Vero o Falso)
let eUnBelFilm = true;

console.log(filmPreferito);
console.log(annoUscita);
console.log(eUnBelFilm);
```

🔍 **Spiegazione dell'esempio:**

- `let filmPreferito = "Ritorno al Futuro";`: Creiamo una stringa. Le virgolette fanno capire a JavaScript che questo è testo puro e semplice, non un comando da eseguire.
    
- `let annoUscita = 1985;`: Creiamo un numero. Non mettiamo le virgolette. In questo modo JavaScript sa che con questa scatola può fare calcoli matematici (come calcolare quanti anni fa è uscito il film).
    
- `let eUnBelFilm = true;`: Creiamo un booleano. Rappresenta una verità. Non servono le virgolette perché `true` e `false` sono parole speciali che JavaScript riconosce da solo.
    

### 🏋️‍♂️ 1.4.3 Esempio pratico (Scheda Fitness)

Immaginiamo di creare il profilo riassuntivo per un utente di un sito web di fitness. Dobbiamo salvare il suo nome, il suo peso e verificare se ha completato l'allenamento di oggi.

```JavaScript
// Dati dell'utente
let nomeUtente = "Sara";      // Stringa
let pesoKg = 62.5;            // Numero (usiamo il punto per i decimali)
let allenamentoFatto = false; // Booleano (inizialmente è falso)

// Il giorno dopo, Sara finisce l'allenamento! Cambiamo l'interruttore su vero
allenamentoFatto = true;

console.log("Nome: " + nomeUtente);
console.log("Peso attuale: " + pesoKg + " kg");
console.log("L'utente ha completato l'allenamento? " + allenamentoFatto);
```

### ✍️ 1.4.4 I Template Literals (Lavorare con i Backtick)

Fino ad ora, per unire testo e variabili, abbiamo usato l'operatore di concatenazione classico `+`. Questo metodo diventa scomodo e genera errori se devi scrivere testi lunghi o andare a capo.

Nello sviluppo moderno si usano i **Template Literals**, racchiudendo il testo tra due backtick ` ` (accenti gravi). All'interno di questi accenti, puoi inserire qualsiasi variabile o calcolo usando la sintassi `${espressione}` e andare a capo liberamente nel file.

```JavaScript
let utente = "Sara";
let pesoKg = 62.5;

// MODO CLASSICO (Scomodo e facile all'errore di spazi)
console.log("Nome: " + utente + " - Peso: " + pesoKg + " kg");

// MODO MODERNO (Template Literal con Backtick)
console.log(`Nome: ${utente} - Peso: ${pesoKg} kg`);
```

### 🔬 1.4.5 I Tipi Primitivi Avanzati e di Mancanza

Oltre ai tre tipi base, JavaScript include quattro primitivi speciali per gestire scenari ingegneristici o la mancanza intenzionale di informazioni:

- **Null (Vuoto Intenzionale):** Rappresenta l'assenza volontaria di un valore. Diciamo esplicitamente al codice: _"Questa variabile esiste, ma ho deciso io di svuotarla o lasciarla vuota per ora"_.
    
- **Undefined (Non Definito):** La dimenticanza. È il valore che JavaScript assegna in automatico a una variabile che è stata creata ma a cui non è ancora stato dato un valore di partenza (es. `let indirizzo;`).
    
- **BigInt:** I numeri standard in JS sono precisi solo fino a circa 9 quadriliardi. Se devi lavorare con numeri astronomici (crittografia o enormi ID di database), crei un BigInt aggiungendo una `n` alla fine del numero.
    
- **Symbol:** Tipo di dato unico e immutabile. Viene usato per creare "chiavi segrete" o identificatori nascosti dentro le strutture dati per evitare che vengano sovrascritti per errore.
    

```JavaScript
let cartaDiCredito = null;      // Null (L'utente non l'ha ancora inserita)
let indirizzoSpedizione;         // Undefined (Variabile creata ma vuota)
const idDatabaseGigante = 9007199254740991n; // BigInt
const chiaveSegreta = Symbol("id");          // Symbol
```

### ⚠️ 1.4.6 Errori comuni

- **Mettere i numeri o booleani tra virgolette per errore:** Se scrivi `let eta = "20";` o `let attivo = "true";`, JavaScript li vede come testo puro. Fare `eta + 1` darà come risultato `"201"` e non `21`, rompendo anche i controlli logici del booleano.
    
- **Usare la virgola nei numeri:** In JavaScript scrivere `let prezzo = 12,50;` provoca un errore di sintassi che blocca il programma. Usa sempre il punto fermo: `let prezzo = 12.50;`.

🚀 **Key Takeaways del Giorno**

- **Le virgolette fanno il testo:** Tutto ciò che si trova tra virgolette è una stringa. Senza virgolette, JavaScript cerca di interpretarlo come numero, booleano o variabile.
    
- **I numeri usano il punto:** Para i numeri con la virgola si usa tassativamente il punto (`.`).
    
- **I Booleani sono interruttori:** Esistono solo due valori possibili, `true` e `false`, perfetti per gestire risposte del tipo sì/no.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                     | **"Spiega Brutta"**                                                  |
| ------------------------- | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| **Stringa**               | Tipo di dato primitivo che rappresenta una sequenza immutabile di caratteri UTF-16.         | Un pezzo di testo racchiuso tra virgolette.                          |
| **Numero (Number)**       | Tipo di dato primitivo numerico a virgola mobile a doppia precisione a 64 bit.              | Qualsiasi valore numerico (intero o decimale) su cui fare calcoli.   |
| **Booleano (Boolean)**    | Tipo di dato logico primitivo che può assumere esclusivamente i valori `true` o `false`.    | L'interruttore del codice: o è ACCESO (`true`) o è SPENTO (`false`). |
| **Null**                  | Valore primitivo speciale che rappresenta l'assenza intenzionale di qualsiasi valore.       | Lo svuotamento volontario di una variabile deciso dal programmatore. |
| **Undefined**             | Valore primitivo assegnato automaticamente a una variabile dichiarata ma non inizializzata. | La dimenticanza: la scatola esiste ma è ancora totalmente vuota.     |
