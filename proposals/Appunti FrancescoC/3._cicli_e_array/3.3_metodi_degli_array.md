
## 🛠️ 3.3 I Metodi degli Array (Base e Avanzati)

Un metodo è una funzione nativa (un comando predefinito) che esegue un'azione specifica sull'array su cui viene agganciato. I metodi si dividono in due grandi categorie: quelli base (per inserire, rimuovere e cercare) e quelli avanzati (per trasformare o filtrare i dati tramite funzioni).

### 🔀 3.3.1 I metodi base di inserimento, rimozione e taglio

I comandi più utilizzati per modificare la struttura dell'array operano sulle estremità o su porzioni specifiche della lista:

- `.`**`push(elemento)`**: Aggiunge uno o più elementi in fondo.
    
- `.`**`pop()`**: Rimuove l'ultimo elemento in fondo e lo restituisce.
    
- `.`**`unshift(elemento)`**: Inserisce uno o più elementi all'inizio (fa scalare gli altri in avanti).
    
- `.`**`shift()`**: Rimuove il primo elemento all'inizio (fa scalare gli altri indietro).
    
- `.`**`splice(indicePartenza, quantiEliminare, nuovoElemento1, ...)`**: Questo metodo è un vero "coltellino svizzero". Serve a rimuovere o sostituire elementi in qualsiasi punto dell'array modificando quello originale.
    
- `.`**`slice(inizio, fine)`**: Copia una parte dell'array e la inserisce in un nuovo array, senza modificare quello originale.
    

```JavaScript
let listaSpesa = ["Pane", "Latte", "Caffè", "Uova"];

// 1. Uso di Splice: eliminiamo 1 elemento a partire dall'indice 1 ("Latte")
listaSpesa.splice(1, 1); // Stato array: ["Pane", "Caffè", "Uova"]

// 2. Uso di Splice per sostituire: all'indice 1 ("Caffè") eliminiamo 1 elemento e mettiamo "Tè"
listaSpesa.splice(1, 1, "Tè");
// Stato array: ["Pane", "Tè", "Uova"]

// 3. Uso di Slice: copiamo gli elementi dall'indice 1 fino all'indice 3 (escluso)
let sottoLista = listaSpesa.slice(1, 3);
// sottoLista contiene: ["Tè", "Uova"] -> listaSpesa rimane identica
```

### 🔍 3.3.2 Metodi di Ricerca e Conversione rapida

Oltre a modificare l'array, spesso abbiamo bisogno di sapere se un elemento esiste o in quale posizione si trova:

- `.`**`includes(valore)`**: Restituisce `true` se il valore è presente nell'array, altrimenti `false`.
    
- `.`**`indexOf(valore)`**: Restituisce l'indice numerico della prima posizione in cui trova il valore. Se il valore non esiste nell'array, restituisce `-1`.
    
- `.`**`join(separatore)`**: Incolla tutti gli elementi dell'array trasformandoli in una singola stringa, separati dal carattere inserito tra le tonde.


```JavaScript
const spesa = ["Pane", "Latte", "Caffè"];

console.log(spesa.includes("Latte")); // Output: true
console.log(spesa.indexOf("Caffè"));  // Output: 2
console.log(spesa.indexOf("Pasta"));  // Output: -1 (non esiste!)

console.log(spesa.join(" - "));      // Output: "Pane - Latte - Caffè"
```

### 🧠 3.3.3 I metodi avanzati di iterazione (Funzionali)

Nello sviluppo moderno, spesso si preferisce evitare il ciclo for classico per fare operazioni comuni come trasformare o filtrare i dati. Per farlo, usiamo metodi avanzati che accettano come argomento una funzione freccia (_arrow function_) che si attiva per ogni singolo elemento dell'array:

- `.`**`forEach()`**: Sostituisce il ciclo for. Passa in rassegna tutti gli elementi uno per uno ed esegue un'azione, ma non restituisce un nuovo array.
    
- `.`**`map()`**: Prende un array, trasforma ogni singolo elemento secondo le tue istruzioni e restituisce un nuovo array modificato, con la stessa lunghezza di quello di partenza.
    
- `.`**`filter()`**: Scorre l'array e trattiene solo gli elementi che superano un controllo logico (una condizione). Restituisce un nuovo array più corto o vuoto.
    
```JavaScript
const prezziEuro = [10, 20, 30, 40];

// 1. .forEach() per stampare i dati
prezziEuro.forEach((prezzo) => console.log("Prezzo: " + prezzo + "€"));

// 2. .map() per raddoppiare tutti i prezzi (crea un nuovo array)
const prezziRaddoppiati = prezziEuro.map((prezzo) => prezzo * 2);
// prezziRaddoppiati: [20, 40, 60, 80]

// 3. .filter() per tenere solo i prezzi maiores di 25
const prezziAlti = prezziEuro.filter((prezzo) => prezzo > 25);
// prezziAlti: [30, 40]
```

### 🛒 3.3.4 Esempio pratico combinato

Immaginiamo la logica di un carrello e-commerce. Abbiamo un array di prezzi di prodotti. Vogliamo applicare l'IVA al 22% a ogni prodotto (usando `.map`) e successivamente vogliamo calcolare il totale filtrando solo i prodotti che costano più di 15€ finiti per un'offerta speciale.

```JavaScript
const prezziBase = [10, 50, 5, 25];

// 1. Trasformiamo i prezzi base aggiungendo l'IVA
const prezziConIva = prezziBase.map((prezzo) => prezzo * 1.22);
// Risultato: [12.2, 61, 6.1, 30.5]

// 2. Filtriamo solo i prodotti "Premium" che superano i 15 euro finiti
const prodottiPremium = prezziConIva.filter((prezzo) => prezzo > 15);
// Risultato: [61, 30.5]

console.log("Prezzi ivati:", prezziConIva);
console.log("Prodotti che superano i 15€:", prodottiPremium);
```

🔍 **Spiegazione dell'esempio:**

- Con `.map((prezzo) => prezzo * 1.22)`, JavaScript prende ogni numero di `prezziBase`, lo moltiplica e salva il risultato in un nuovo array chiamato `prezziConIva`.
    
- Con `.filter((prezzo) => prezzo > 15)`, il programma prende l'array appena creato, scarta 12.2 e 6.1 perché non soddisfano la condizione, e genera l'array finale `prodottiPremium`.
    

### ⚠️ 3.3.5 Errori comuni

- **Confondere .splice() con .slice():** I loro nomi si somigliano, ma fanno cose opposte. `.splice()` (con la 'p') modifica l'array originale, distruggendo o cambiando i suoi dati sul posto. `.slice()` (senza 'p') è sicuro, non tocca l'array originale e si limita a fare una copia di una porzione.
    
- **Dimenticare il return implicito nelle arrow function di .map() o .filter():** Se scrivi `const nuovi = array.map((el) => { el * 2 })` usando le parentesi graffe senza scrivere la parola chiave `return`, il nuovo array si riempirà di valori `undefined`. Se usi le parentesi graffe devi mettere `return`. Se non le metti, il valore viene restituito da solo in automatico: `array.map((el) => el * 2)`.
    
- **Pensare che .forEach() restituisca qualcosa:** Scrivere `let risultato = array.forEach(...)` salverà sempre `undefined` dentro la variabile. Il `forEach` serve solo ad eseguire azioni (come i `console.log`), non a generare nuovi dati o modificare l'array per l'assegnazione.
    

## 📌 3.4 Approfondimento leggero: Modificare l'array originale o crearne uno nuovo? (Mutabilità)

In JavaScript avanzato è fondamentale distinguere tra metodi mutabili (che modificano i dati di partenza) e immutabili (che lasciano intatto l'array iniziale e ne creano uno nuovo).

- **Metodi Mutabili (Attenzione!):** `push()`, `pop()`, `shift()`, `unshift()`, `splice()`.
    
- **Metodi Immutabili (Più sicuri):** `slice()`, `map()`, `filter()`.
    

Nello sviluppo moderno (specialmente quando si utilizzano librerie come React) si preferisce quasi sempre l'approccio immutabile per evitare che una funzione modifichi per errore dati usati in altre parti del programma.

🚀 **Key Takeaways del Giorno**

- **Splice taglia, Slice copia:** Usa `.splice()` se devi asportare o iniettare elementi modificando la lista originale. Usa `.slice()` per fare una foto parziale senza fare danni.
    
- **Map trasforma:** `.map()` è il re delle trasformazioni. Genera sempre un array identico per numero di elementi, ma con i valori modificati uno per uno.
    
- **Filter seleziona:** `.filter()` riduce le dimensioni della lista trattenendo solo gli elementi che superano il tuo test logico.
