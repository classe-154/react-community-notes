## ⚖️ 2.2 Operatori di confronto

### 📝 2.2.1 Gli Operatori Relazionali

Per fare in modo che JavaScript possa calcolare concretamente queste risposte, utilizziamo gli operatori relazionali (chiamati anche operatori di confronto).

Il loro compito è mettere in relazione due elementi (variabili o valori fissi) e confrontarli. Il risultato di questo confronto è sempre e solo un valore booleano.

I simboli fondamentali da conoscere sono:

- `>` (Maggiore) e `<` (Minore)
    
- `>=` (Maggiore o uguale) e `<=` (Minore o uguale)
    
- `===` (Strettamente uguale)
    
- `!==` (Diverso da)
    

Senza gli operatori relazionali, il computer non sarebbe in grado di capire se hai inserito la password corretta, se il prezzo di un prodotto è scontato o se hai abbastanza soldi sulla carta per completare un acquisto.

### 💻 2.2.2 Esempio base

Vediamo come funzionano i confronti numerici diretti e come JavaScript salva il loro risultato.

```JavaScript
let altezzaUtenteCm = 175;
let altezzaMinimaGiostraCm = 140;

// Confrontiamo se l'altezza dell'utente supera o eguaglia quella minima
let puoSalireSullaGiostra = altezzaUtenteCm >= altezzaMinimaGiostraCm;

console.log(puoSalireSullaGiostra); // Stamperebbe true
```

🔍 **Spiegazione dell'esempio:**

- Creiamo due variabili numeriche separate.
    
- Nella riga del confronto, JavaScript sostituisce i nomi delle variabili con i loro valori, leggendo: `175 >= 140`.
    
- Poiché 175 è effettivamente più grande di 140, l'operatore `>=` calcola come risultato `true`.
    
- Questo valore booleano viene memorizzato nella variabile `puoSalireSullaGiostra`.
    

### 🛠️ 2.2.3 Esempio pratico

Immaginiamo il sistema di controllo di un conto bancario online. Dobbiamo verificare se il PIN inserito è corretto e se il saldo è sufficiente per coubrire la cifra che l'utente vuole prelevare.

```JavaScript
// Dati memorizzati nel database della banca
const pinCorretto = "4321";
let saldoDisponibile = 250;

// Dati digitati dall'utente allo sportello
let pinInserito = "4321";
let cifraDaPrelevare = 300;

// 1. Controllo di uguaglianza per il PIN (confronto tra testi)
if (pinInserito === pinCorretto) {
  console.log("PIN corretto. Verifica del saldo in corso...");
  
  // 2. Controllo di grandezza per il denaro (confronto tra numeri)
  if (cifraDaPrelevare <= saldoDisponibile) {
    console.log("Prelievo autorizzato! Prendi i contanti.");
  } else {
    console.log("Errore: Non hai abbastanza soldi sul conto.");
  }

} else {
  console.log("Errore: Il PIN inserito è sbagliato.");
}
```

🔍 **Spiegazione dell'esempio:**

- Il primo `if` usa l'operatore `===` per vedere se la stringa `pinInserito` è identica a `pinCorretto`. Essendo uguali (`"4321"`), entriamo nel blocco.
    
- Dentro troviamo un secondo `if` nidificato: `if (cifraDaPrelevare <= saldoDisponibile)`. JavaScript valuta se `300 <= 250`.
    
- Il calcolo dà `false`. Il programma salta la conferma e corre nella stanza dell'else interna, stampando il messaggio di saldo insufficiente.
    

### ⚠️ 2.2.4 Errori comuni

- **Usare l'operatore debole == al posto di ===:** In JavaScript esiste anche il doppio uguale (`==`), ma non si usa perché fa conversioni automatiche pericolose. Ad esempio, `5 == "5"` per lui è `true` (unisce il numero al testo). L'operatore stretto `===` controlla anche il tipo di dato, quindi `5 === "5"` restituisce correttamente `false`.
    
- **Confondere l'ordine dei simboli in >= e <=:** Scrivere `=>` o `=<` per fare un confronto blocca il programma con un errore di sintassi. Il simbolo di uguale va scritto sempre dopo il beccuccio.
    
- **Dimenticare che il testo è sensibile alle maiuscole:** Se confronti `"Admin" === "admin"`, il risultato sarà `false`. JavaScript confronta i caratteri esatti, e la "A" maiuscola ha un valore informatico diverso dalla "a" minuscola.
    

## 📌 2.3 Approfondimento leggero: Il confronto tra tipi diversi (Type Coercion)

Quando usi gli operatori di confronto come `>`, `<`, `>=` o `<=`, JavaScript cerca sempre di essere d'aiuto. Se provi a confrontare un numero vero con una stringa che contiene un numero, lui trasforma temporaneamente la stringa in numero per non rompere il programma:

```JavaScript
let numeroVero = 10;
let testoNumero = "5";

console.log(numeroVero > testoNumero); // Restituisce true!
```

Anche se il programma non si rompe, è sempre una buona abitudine confrontare dati dello stesso tipo (numeri con numeri, testi con testi) per evitare comportamenti imprevedibili.

🚀 **Key Takeaways del Giorno**

- **Producono solo Booleani:** Gli operatori relazionali non modificano mai i dati, servono solo a generare una risposta `true` o `false`.
    
- **Usa sempre tre uguali (===):**  È l'unico modo sicuro per verificare l'uguaglianza in JavaScript, perché controlla sia il valore che il tipo di dato.
    
- **L'ordine dei simboli è fisso:** Nei simboli combinati come `>=` e `!==`, l'uguale si posiziona sempre alla fine della combinazione.
    

📖 **Glossario**

| **Termine Istituzionale**        | **Definizione Formale**                                                       | **"Spiega Brutta"**                                                                   |
| -------------------------------- | ----------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **Operatore Relazionale**        | Simbolo logico-matematico che definisce la relazione tra due valori.          | Lo strumento per paragonare due variabili e vedere chi vince o se si somigliano.      |
| **=== (Uguaglianza Stretta)**    | Confronto che verifica l'identità di valore e di tipo senza conversioni.      | Il controllo doganale pignolo: i due dati devono essere identici e dello stesso tipo. |
| **!== (Disuguaglianza Stretta)** | Operatore che restituisce `true` se due valori differiscono in valore o tipo. | Il controllo del "sono diversi?": risponde sì se i dati non si somigliano affatto.    |
