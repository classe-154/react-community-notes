## 🔄 5.4 Ciclo For e DOM (Iterazione Massiva sulle Collezioni)

Nello sviluppo reale, capita raramente di dover modificare un singolo elemento isolato tramite ID. Molto più spesso un applicativo deve agire su intere collezioni di elementi omogenei: applicare uno stile a tutte le righe di una tabella, nascondere una serie di schede prodotto o agganciare un ascoltatore di eventi a tutti i bottoni di una galleria fotografica.

Per compiere queste operazioni in blocco si utilizzano i costrutti iterativi applicati alle **NodeList** (le collezioni statiche restituite da `querySelectorAll`). Ciclare il DOM significa scorrere i nodi uno ad uno, estrarre il singolo _Element Object_ ed eseguire la logica di manipolazione richiesta.

### 🧱 5.4.1 Iterazione Standard con il Ciclo For Classico

Il ciclo `for` classico rappresenta lo strumento più efficiente, universale e controllabile per scorrere una NodeList. Funziona esattamente come con i normali array: si sfrutta la proprietà `.length` della collezione per definire il limite del contatore e si accede al singolo elemento usando le parentesi quadre con l'indice `[i]`.

```JavaScript
// Selezioniamo tutte le schede prodotto presenti nella pagina
const tutteLeSchede = document.querySelectorAll(".txt-scheda-prodotto");

// Cicliamo la collezione dal primo all'ultimo elemento
for (let i = 0; i < tutteLeSchede.length; i++) {
  // Estraiamo il singolo Element Object corrispondente all'indice corrente
  const schedaCorrente = tutteLeSchede[i];
  
  // Modifichiamo l'elemento in blocco aggiungendo una classe CSS
  schedaCorrente.classList.add("caricato");
}
```

### 🚀 5.4.2 Iterazione Moderna con forEach

A differenza delle vecchie collezioni, le moderne NodeList implementano nativamente il metodo `.forEach()`. Questo permette di scorrere gli elementi nello script esecutivo usando una sintassi più snella basata su una _Arrow Function_ di callback, senza dover gestire manualmente gli indici.

```JavaScript
const tuttiIParagrafi = document.querySelectorAll("p.testo-riassunto");

// Il forEach passa automaticamente il singolo elemento come primo argomento della callback
tuttiIParagrafi.forEach(paragrafo => {
  paragrafo.style.color = "#333333";
});
```

#### ⚡ Il caso d'uso più comune: Eventi di gruppo

Il motivo principale per cui ciclerai il DOM è l'esigenza di applicare un comportamento (un `addEventListener`) a molti elementi simili contemporaneamente, come una lista di pulsanti "Elimina" o le voci di un menu di navigazione:

```JavaScript
const tuttiIBottoniElimina = document.querySelectorAll(".btn-elimina");

// Giriamo la lista dei bottoni
tuttiIBottoniElimina.forEach(bottone => {
  // Agganciamo l'ascoltatore a OGNI singolo bottone durante il ciclo
  bottone.addEventListener("click", (event) => {
    console.log("Hai cliccato su un pulsante specifico!");
    // Possiamo usare evento.target per capire quale riga specifica gestire
  });
});
```

### ⏱️ 5.4.3 Concetto Avanzato: Caching della Lunghezza e Performance

Quando si esegue un ciclo `for` classico sul DOM scrivendo `i < tutteLeSchede.length`, il browser compie un'operazione microscopica ma ripetitiva: ad ogni singolo giro del ciclo va a interrogare la proprietà `.length` direttamente dall'oggetto presente in memoria RAM. Se la NodeList contiene centinaia di elementi, questo controllo continuo può impattare sulle performance generali.

La tecnica ingegneristica del **Caching della Lunghezza** risolve questo problema. Si memorizza la lunghezza della collezione in una variabile di supporto locale all'interno dell'inizializzazione del ciclo stesso. In questo modo, il browser legge il valore una volta sola all'avvio e lo riutilizza all'istante per tutti i giri successivi.

```JavaScript
const vociMenu = document.querySelectorAll(".voce-navigazione");

// Ottimizzazione: salviamo la lunghezza in 'totaleVoci' durante l'inizializzazione del ciclo
for (let i = 0, totaleVoci = vociMenu.length; i < totaleVoci; i++) {
  vociMenu[i].classList.remove("attivo");
}
```

### 🔀 5.4.4 Trasformare una NodeList in un vero Array

Sebbene la NodeList supporti il ciclo `.forEach()`, essa rimane un "Array-like object" e non possiede i metodi nativi di trasformazione degli array come `.map()` o `.filter()`.

Se abbiamo bisogno di filtrare gli elementi del DOM in base a una determinata caratteristica prima di modificarli, dobbiamo convertire la NodeList in un vero array sfruttando lo **Spread Operator (`...`)**:

```JavaScript
const tuttiIBottoni = document.querySelectorAll(".btn-azione");

// Trasformiamo la NodeList statica in un vero Array JavaScript
const arrayBottoni = [...tuttiIBottoni];

// Ora possiamo usare liberamente i metodi di filtraggio avanzati
const bottoniDisabilitati = arrayBottoni.filter(bottone => {
  return bottone.hasAttribute("disabled");
});

// Eseguiamo una modifica mirata solo sugli elementi risultanti dal filtro
bottoniDisabilitati.forEach(bottone => {
  bottone.style.opacity = "0.5";
});
```

### ⚠️ 5.4.5 Errori comuni

- **Applicare modifiche massive direttamente sulla NodeList:** Scrivere `document.querySelectorAll(".scheda").style.display = "none"` causerà un errore bloccante di runtime (`TypeError`). Devi ricordarti che la NodeList è il "recinto" (il contenitore) che racchiude gli elementi, non l'elemento stesso. L'azione di modifica va applicata sul singolo elemento estratto all'interno del ciclo (`lista[i]` o il parametro della callback).
    
- **Il bug del "Salto dell'Indice" con collezioni Live:** Se usi un ciclo `for` classico su una collezione _Live_ (restituita da metodi obsoleti come `getElementsByClassName`) e all'interno del ciclo rimuovi la classe usata per la selezione, l'elemento esce istantaneamente dalla collezione in tempo reale. Questo accorcia la lunghezza totale della lista a ogni giro, provocando lo slittamento degli elementi rimasti e facendo saltare al ciclo il controllo del nodo successivo.

🚀 **Key Takeaways del Giorno**

- **Le NodeList si ciclano:** Usa il ciclo `for` classico (obbligatorio in `function.js` per il nostro metodo didattico) o il metodo `.forEach()` per applicare modifiche massive a gruppi di tag omogenei.
    
- **Ottimizza i cicli:** Nelle iterazioni su larga scala, memorizza la lunghezza della collezione in una variabile d'appoggio per risparmiare preziose risorse di calcolo del browser.
    
- **Converti per filtrare:** Se devi scremare o manipolare gli elementi del DOM usando `.filter()` o `.map()`, ricordati di trasformare preventivamente la NodeList in un vero Array utilizzando lo Spread operator `[...]`.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                                                | **"Spiega Brutta"**                                                                                                                      |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **Iterazione del DOM**    | Processo sequenziale di scorrimento e manipolazione dei nodi contenuti all'interno di una collezione del DOM.          | Fare un giro di ricognizione su una lista di tag agganciati per cambiarli tutti quanti uno dopo l'altro.                                 |
| **Array-like Object**     | Oggetto JavaScript che possiede una proprietà `length` e indici numerici, ma non eredita i metodi del prototipo Array. | Un finto array: ha una lunghezza e gli elementi numerati come una lista, ma non possiede i metodi nativi evoluti (es. `map` e `filter`). |
| **Caching**               | Pratica ingegneristica di memorizzazione temporanea di un dato calcolato o letto per velocizzare gli accessi futuri.   | Salvare un valore dentro una variabile locale vicina per evitare che il computer vada a cercarselo in memoria profonda a ogni istante.   |
