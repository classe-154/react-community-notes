## рџ’Ћ 5.2 Element Object (Selezione e ProprietГ  degli Elementi)

Ogni tag HTML presente nella pagina web, una volta tradotto dal browser all'interno del DOM, diventa un oggetto software di tipo **Element (Element Object)**. In quanto oggetto, esso eredita un set ricchissimo di proprietГ  e metodi nativi che permettono a JavaScript di eseguire due operazioni fondamentali: trovare l'elemento all'interno della mappa della pagina (**Selezione**) e leggerne o modificarne il contenuto (**Manipolazione**).

### рџ”Ќ 5.2.1 Metodi Standard di Selezione

Prima di poter modificare un elemento, dobbiamo dire a JavaScript come trovarlo. Lo standard moderno mette a disposizione due approcci: i metodi selettivi classici e i metodi universali basati sui selettori CSS dello stile.

```JavaScript

const contenitorePrincipale = document.getElementById("contenitore-principale");

// 2. Selezionare il PRIMO elemento che corrisponde a un selettore CSS (Universale e Standard)
const primoTitolo = document.querySelector(".titolo-sezione");

// 3. Selezionare TUTTI gli elementi che corrispondono a un selettore CSS (Restituisce una collezione)
const tuttiIParagrafi = document.querySelectorAll("p.testo-descrittivo");
```

### вњЌпёЏ 5.2.2 Manipolazione dei Contenuti: textContent vs innerHTML

Una volta ottenuto il riferimento all'oggetto Element, possiamo accedere al suo testo o al suo codice interno tramite due proprietГ  fondamentali, che agiscono in modo radicalmente diverso sulla sicurezza e sulla struttura della pagina:

- **`textContent`:** Legge o inserisce esclusivamente **testo puro**. Se provi a passare del codice HTML (es. `<strong>testo</strong>`), lo tratterГ  come testo letterale, mostrando i tag espliciti a schermo senza modellarli. Г€ la scelta piГ№ sicura per evitare attacchi informatici di tipo XSS (_Cross-Site Scripting_).
    
- **`innerHTML`:** Legge o inserisce vero e proprio **codice HTML**. Il browser eseguirГ  il parsing dei tag passati, ricostruendo l'albero dei nodi all'interno di quell'elemento.
    

```JavaScript

// Uso di textContent (Sicuro, interpreta tutto come testo piatto)
scatolaInformativa.textContent = "Attenzione: operazione completata.";

// Uso di innerHTML (Modifica la struttura inserendo nuovi tag HTML)
scatolaInformativa.innerHTML = "Attenzione: <strong>operazione completata</strong>.";
```

### рџЋЁ 5.2.3 Manipolazione degli Attributi e delle Classi

Un Element Object permette di gestire al volo i suoi attributi HTML (come `src`, `href`, `id`) e le sue classi stilistiche CSS, senza dover riscrivere o distruggere l'intero tag.

```JavaScript

// 1. Gestione Attributi standard

let urlCorrente = linkEsterno.getAttribute("href");      // Legge il valore dell'attributo

// 2. Gestione delle Classi CSS tramite l'oggetto nativo classList

schedaProdotto.classList.add("evidenziato");    // Aggiunge una classe CSS
schedaProdotto.classList.remove("attivo");      // Rimuove una classe CSS
schedaProdotto.classList.toggle("nascondi");    // Se la classe c'ГЁ la toglie, se manca la mette
```

### рџ“¦ 5.2.4 Collezioni di Nodi (Live vs Static NodeList)

Quando chiedi a JavaScript di prendere piГ№ elementi insieme (ad esempio usando `querySelectorAll`), il motore del browser non ti restituisce un classico Array di dati, ma una collezione speciale chiamata **NodeList** (o **HTMLCollection** a seconda dei metodi usati).

A livello ingegneristico, c'ГЁ una differenza cruciale su come queste strutture gestiscono la memoria e i dati:

|**Tipo di Collezione**|**Come viene generata**|**Comportamento in Memoria**|**Impatto Performance**|
|---|---|---|---|
|**Collezioni Vive (Live)**|Metodi vecchi (`getElementsByClassName`, `getElementsByTagName`)|**Connesse in tempo reale.** Se un elemento viene rimosso dal DOM un millisecondo dopo, la lista si aggiorna e si accorcia da sola.|**Pesante.** Il browser deve monitorare continuamente la pagina.|
|**Collezioni Statiche (Static)**|Metodo moderno (`querySelectorAll`)|**Istantanea fissa (Screenshot).** Se la pagina cambia in seguito, la lista memorizzata rimane invariata.|**Leggero.** Г€ il sistema standard e prevedibile di oggi.|

La risposta breve ГЁ: **No, ГЁ un "finto" array (un oggetto Array-like).** * **вњ… Cosa PUOI fare:** Puoi usare `.length` per contare gli elementi, puoi usare le staffe quadre numeriche (es. `lista[0]`) per estrarre il singolo elemento e puoi usare il ciclo `.forEach()` per applicare modifiche su ciascun nodo.

- **вќЊ Cosa NON puoi fare:** Non hai i metodi nativi degli array veri. Non puoi usare `.push()` o `.pop()` per inserire o togliere elementi, e non puoi usare metodi di filtraggio o trasformazione complessi.
    

#### рџ’» L'Esempio Pratico e il "Trucco del Mago"

Immagina di avere questa lista di bottoni in HTML:


Se nel file JavaScript provi a filtrare direttamente la NodeList, l'applicazione si bloccherГ . Per risolvere il limite, applichiamo lo **Spread Operator (`...`)** per convertire la collezione in un array reale al 100%:

```JavaScript

// вќЊ ERRORE CLASSICO: Questo romperГ  il programma

// Errore: TypeError: nodiBottoni.filter is not a function

// рџ’Ў APPLICHIAMO IL TRUCCO DEL MAGO (Trasformazione in Array vero con lo Spread)

// Ora che ГЁ un array nativo, possiamo usare i metodi avanzati!

```

### вљ пёЏ 5.2.5 Errori comuni

- **Dimenticare che `querySelectorAll` restituisce un contenitore:** Se scrivi `document.querySelectorAll(".bottone").classList.add("attivo")`, il codice andrГ  in errore lanciando un `TypeError`. Non puoi applicare `classList` o `textContent` direttamente sull'intera NodeList. Devi prima estrarre il singolo elemento indicizzandolo (es. `[0]`) o ciclare la collezione.
    
- **Abusare di `innerHTML` per inserire testi semplici:** Sostituire sistematicamente `textContent` con `innerHTML` costringe il browser a distruggere e ricostruire i nodi interni ogni volta, rallentando l'applicazione e aprendo pericolose falle di sicurezza (XSS) se i dati inseriti provengono da form compilati dagli utenti.
