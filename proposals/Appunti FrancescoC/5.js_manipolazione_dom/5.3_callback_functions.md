## 🔌 5.3 Callback Functions e Gestione Eventi

Una pagina web non è un elemento statico; è un ambiente interattivo in cui l'utente compie continuamente azioni: clicca su un bottone, digita del testo, muove il mouse o invia un modulo. JavaScript intercetta queste azioni grazie al concetto di **Evento** e risponde eseguendo del codice specifico attraverso le **Callback Functions**.

A livello teorico, una **Callback** è una funzione che viene passata come argomento (ingrediente) dentro un'altra funzione, per essere "chiamata all'indietro" (eseguita) solo quando si verifica una determinata condizione o azione temporale.

### 📡 5.3.1 Il Meccanismo degli Eventi: addEventListener

Il metodo standard e professionale per agganciare un evento a un elemento del DOM è `addEventListener`. Questo metodo richiede due argomenti fondamentali:

1. Il nome dell'evento sotto forma di stringa (es. `"click"`, `"submit"`, `"input"`, `"keydown"`).
    
2. La funzione di Callback da eseguire automaticamente ogni volta che l'evento si scatena.
    

```JavaScript
const bottoneInvia = document.querySelector(".btn-invia");

// Definizione della funzione di Callback (Sintassi Regular)
function gestisciClick() {
  console.log("Il bottone è stato cliccato con successo!");
}

// Aggancio dell'evento: passiamo il nome della funzione SENZA parentesi tonde
bottoneInvia.addEventListener("click", gestisciClick);

// Equivalente moderno con Arrow Function anonima direttamente come Callback:
bottoneInvia.addEventListener("click", () => {
  console.log("Esecuzione immediata tramite funzione a freccia.");
});
```

### 🎯 5.3.2 L'Oggetto Evento (event) e i suoi Metodi Nativi

Ogni volta che si scatena un evento, il browser crea automaticamente un oggetto software ricchissimo di informazioni e lo passa come primo argomento implicito alla funzione di callback. Questo oggetto viene comunemente identificato con il nome di parametro `e` oppure `event`.

L'oggetto contiene proprietà utili (come `event.target`, che indica l'elemento esatto del DOM che ha subito l'azione) e metodi di controllo ingegneristici fondamentali:

- **`event.preventDefault()`:** Blocca il comportamento predefinito e nativo che il browser eseguirebbe a seguito di quell'azione. L'uso più comune è nei form web per impedire il caricamento e il refresh automatico della pagina durante l'invio.
    
- **`event.stopPropagation()`:** Interrompe la propagazione dell'evento verso gli elementi genitori dell'albero del DOM.
    

```JavaScript
const moduloContatti = document.querySelector("#form-contatti");

moduloContatti.addEventListener("submit", (event) => {
  // 1. Blocchiamo il refresh della pagina nativo del form
  event.preventDefault(); 
  
  // 2. Leggiamo l'elemento esatto che ha generato l'evento
  console.log("Form intercettato tramite:", event.target);
  console.log("I dati sono pronti per essere elaborati in sicurezza.");
});
```

#### ⌨️ Intercettare la tastiera: Leggere i campi di testo

Quando l'ascolto è posizionato su una casella di testo (`<input>`) usando l'evento `"input"`, l'oggetto evento diventa lo strumento per catturare ciò che l'utente scrive. Sfruttando la proprietà `evento.target.value`, otteniamo la stringa in tempo reale:

```JavaScript
const casellaEmail = document.querySelector(".input-email");

casellaEmail.addEventListener("input", (event) => {
  // 'evento.target' è la casella stessa, '.value' è il testo scritto dentro in quel momento
  console.log("L'utente sta digitando: ", event.target.value);
});
```

### 🫧 5.3.3 Concetto Avanzato: Propagazione degli Eventi (Bubbling vs Capturing)

Quando un utente clicca su un elemento del DOM, non sta cliccando solo su quel singolo tag, ma si trova contemporaneamente sopra a tutti i suoi contenitori genitori fino ad arrivare alla radice `window`. JavaScript gestisce questa situazione attraverso la **Propagazione dell'Evento (Event Propagation)**, che si sviluppa in due fasi opposte:

- **Event Capturing (Fase di Cattura):** L'evento scende dall'alto verso il basso. Parte dall'oggetto `window` e attraversa tutti i tag genitori fino a raggiungere l'elemento esatto cliccato (il `target`). È una fase utilizzata raramente nello sviluppo comune.
    
- **Event Bubbling (Fase di Bolla):** È il comportamento predefinito di JavaScript. Una volta raggiunto l'elemento target, l'evento risale verso l'alto come una bolla d'aria nell'acqua, attivando in sequenza tutti gli `addEventListener` dello stesso tipo agganciati sui tag genitori.
    

```JavaScript
const sezioneEsterna = document.querySelector(".contenitore-macro");
const bottoneInterno = document.querySelector(".btn-micro");

sezioneEsterna.addEventListener("click", () => {
  console.log("1. Click intercettato dal CONTENITORE macro.");
});

bottoneInterno.addEventListener("click", (event) => {
  console.log("2. Click intercettato dal BOTTONE micro.");
  // Se non vogliamo che la "bolla" salga ad attivare anche la sezioneEsterna, decommentiamo la riga sotto:
  // evento.stopPropagation();
});

// Risultato del click sul bottone: la console stamperà PRIMA il punto 2 e SUBITO DOPO il punto 1 a causa del Bubbling.
```

### ⚠️ 5.3.4 Errori comuni

- **Invocare la Callback dentro addEventListener:** Scrivere `bottone.addEventListener("click", gestisciClick())` (con le parentesi tonde) è un errore grave. Le tonde eseguono la funzione immediatamente durante il parsing, assegnando il risultato del `return` all'evento. Al momento del click dell'utente non succederà nulla. La callback va passata solo tramite il suo **nome di riferimento** (senza tonde), delegando al browser il compito di invocarla al momento opportuno.
    
- **Dimenticare il `preventDefault` nei form:** Non inserire il `preventDefault` durante l'ascolto dell'evento `"submit"` farà sì che la pagina si ricarichi istantaneamente. Questo cancella tutte le variabili locali e i dati inseriti dall'utente prima ancora che JavaScript possa elaborarli o salvarli.

🚀 **Key Takeaways del Giorno**

- **Le Callback sono reattive:** Una callback legata a un evento non viene eseguita subito, ma rimane in ascolto dormiente finché l'utente non compie l'azione prefissata.
    
- **Passa il riferimento, non l'esecuzione:** Quando agganci una funzione a un listener, scrivila rigorosamente senza parentesi tonde per evitare che parta prima del tempo.
    
- **Gli eventi risalgono la corrente:** Per impostazione predefinita l'evento risale l'albero del DOM (`Bubbling`). Usa `evento.stopPropagation()` se vuoi isolare l'azione impedendo di attivare i listener dei padri.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                                       | **"Spiega Brutta"**                                                                                                             |
| ------------------------- | ------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **Callback Function**     | Funzione passata come argomento all'interno di un'altra funzione per essere eseguita in un secondo momento.   | Una funzione "post-it": la lasci scritta lì e dici al computer di eseguirla solo quando si verifica una determinata condizione. |
| **Event Listener**        | Oggetto o metodo che rimane in ascolto di uno specifico segnale hardware o software generato sulla pagina.    | Una sentinella che sorveglia un tag della pagina e scatta non appena l'utente fa un click o digita sulla tastiera.              |
| **Event Bubbling**        | Processo di propagazione in cui l'evento risale progressivamente dal nodo target verso tutti i nodi antenati. | L'effetto domino verso l'alto: se clicchi sul tag figlio, l'evento si attiva di riflesso anche sul tag padre e sul nonno.       |
| **Event Target**          | Riferimento all'oggetto del DOM che ha originato l'evento durante la fase di attivazione.                     | Il bersaglio colpito: il tag preciso della pagina su cui l'utente ha materialmente cliccato o ha iniziato a digitare.           |
