## ⚙️ 4.2 Sintassi Regular (Parametri, Argomenti e Return)

La sintassi _regular_ (standard) di JavaScript permette di creare un canale di comunicazione bidirezionale con la funzione: possiamo inviare dei dati in ingresso, lavorarli e sputare fuori un risultato finale in uscita.

### 📥 4.2.1 I Dati in Ingresso: Parametri vs Argomenti

Spesso questi due termini vengono confusi, ma indicano due elementi specifici in momenti temporali diversi:

- **I Parametri (I Segnaposto):** Sono i nomi delle variabili che inventi e scrivi dentro le parentesi tonde nella fase di definizione della funzione. Esistono solo sulla carta e fungono da variabili locali temporanee.
    
- **Gli Argomenti (I Valori Reali):** Sono i valori reali e concreti (stringhe, numeri, booleani) che passi alla funzione nel momento esatto dell'invocazione.
    

```JavaScript
// 'nome' e 'ruolo' sono PARAMETRI (segnaposto vuoti)
function generareBadge(nome, ruolo) {
  console.log(`Profilo: ${nome} - Ruolo: ${ruolo}`);
}

// "Alice" e "Developer" sono gli ARGOMENTI reali passati dentro i segnaposto
generareBadge("Alice", "Developer"); 
```

### 📤 4.2.2 I Dati in Uscita: L'istruzione Return

Una funzione che si limita a stampare in console con `console.log` è come una fabbrica che produce un oggetto e lo lascia sul pavimento del magazzino. Per poter riutilizzare il risultato di una funzione nel resto del nostro programma, dobbiamo usare la parola chiave `return`.

Il `return` fa due cose fondamentali:

1. **Esporta il valore:** Prende il dato calcolato dentro la funzione e lo "sputa fuori", permettendoci di salvarlo in una variabile esterna.
    
2. **Interrompe l'esecuzione:** Funziona come un interruttore di sicurezza. Nel momento esatto in cui JavaScript incontra un `return`, esce immediatamente dalla funzione. Qualsiasi riga di codice scritta sotto il `return` non verrà mai letta.
    

```JavaScript
function calcolaSconto(prezzoIniziale) {
  let sconto = prezzoIniziale * 0.2;
  let prezzoFinale = prezzoIniziale - sconto;
  
  return prezzoFinale; // Il risultato viene inviato all'esterno
  
  console.log("Questo testo è invisibile"); // Errore logico: codice irraggiungibile
}

// Riceviamo il valore del return e lo salviamo in una variabile utile
let prezzoScontatoTotale = calcolaSconto(150);
console.log(`Prezzo finale da pagare: ${prezzoScontatoTotale}€`);
```

#### 🚪 Il Return implicito e l'Uscita Anticipata (Early Exit)

Ci sono due comportamenti "silenziosi" del `return` che devi conoscere:

- **Se ti dimentichi il return:** Se una funzione esegue dei calcoli ma non ha un'istruzione `return`, JavaScript restituisce in automatico il valore `undefined`.
    
- **Il return come "freno a mano" (Early Exit):** Puoi usare la parola chiave `return;` da sola, senza affiancarle nessun valore. Serve nei controlli iniziali per bloccare immediatamente il sotto-programma se i dati in ingresso non sono validi, evitando di eseguire il resto del codice inutilmente.
    


```JavaScript
function approvaAccesso(eta) {
  if (eta < 18) {
    console.log("Accesso negato.");
    return; // Blocca la funzione qui ed esce! Il codice sotto non viene eseguito
  }
  
  console.log("Benvenuto nel pannello di controllo!");
  // ... altro codice per utenti maggiorenni
}
```

### ⚙️ 4.2.3 Concetto Avanzato 1: Parametri di Default

Cosa succede se definiamo una funzione con due parametri, ma durante l'invocazione passiamo un solo argomento? JavaScript non blocca il programma, ma assegna in automatico il valore `undefined` al parametro mancante, rischiando di generare calcoli errati o `NaN`.

Per evitare questo problema, la sintassi regular moderna permette di assegnare un **Valore di Default** direttamente nella firma della funzione usando l'operatore `=`. Se l'argomento viene passato, il default viene sovrascritto; se viene dimenticato, entra in azione il valore di salvataggio.

```JavaScript
// Se non viene passato il moltiplicatore, il sistema userà 1 in automatico
function calcolaPunteggio(puntiBase, moltiplicatore = 1) {
  return puntiBase * moltiplicatore;
}

console.log(calcolaPunteggio(50));    // Manca il secondo argomento -> 50 * 1 = 50
console.log(calcolaPunteggio(50, 3)); // Passato il secondo argomento -> 50 * 3 = 150
```

### 🧠 4.2.4 Concetto Avanzato 2: Passaggio per Valore vs Passaggio per Riferimento

Questo è uno dei comportamenti ingegneristici più importanti di JavaScript e dipende dal tipo di dato che passi come argomento:

- **Tipi Primitivi (Numeri, Stringhe, Booleani):** Vengono passati per **Valore**. La funzione riceve una copia esatta del dato. Se modifichi il parametro dentro la funzione, la variabile originale esterna rimane intatta.
    
- **Tipi Complessi (Oggetti, Array):** Vengono passati per **Riferimento**. La funzione non riceve una copia, ma l'indirizzo di memoria del dato originale. Se modifichi l'oggetto o l'array dentro la funzione, modificherai istantaneamente anche l'originale all'esterno.
    

```JavaScript
// 1. Passaggio per VALORE (Dato protetto)
let punteggioEsterno = 10;

function modificaNumero(numero) {
  numero = 20; // Cambia solo la copia locale
}

modificaNumero(punteggioEsterno);
console.log(punteggioEsterno); // Rimane 10!

// 2. Passaggio per RIFERIMENTO (Dato condiviso)
let utenteEsterno = { nome: "Marco" };

function modificaOggetto(oggetto) {
  oggetto.nome = "Luca"; // Modifica direttamente la memoria centrale
}

modificaOggetto(utenteEsterno);
console.log(utenteEsterno.nome); // È diventato "Luca"!
```

### ⚠️ 4.2.5 Errori comuni

- **Confondere console.log con return:** Il `console.log` serve solo a noi sviluppatori per vedere un dato sul monitor, ma non restituisce nulla al flusso logico del programma. Se scrivi `let x = miaFunzione();` e dentro la funzione c'è solo un `console.log`, la variabile `x` sarà valorizzata come `undefined`.
    
- **Invertire l'ordine degli argomenti:** JavaScript associa gli argomenti ai parametri esclusivamente in base all'ordine posizionale in cui li scrivi. Se la funzione è `function iscrizione(nome, email)` e tu la invochi scrivendo `iscrizione("info@email.com", "Matteo")`, il sistema salverà l'indirizzo email dentro il parametro `nome` e viceversa.

🚀 **Key Takeaways del Giorno**

- **I parametri sono scatole, gli argomenti il contenuto:** I parametri si definiscono una volta sola nella firma della funzione, gli argomenti cambiano dinamicamente ad ogni invocazione.
    
- **Il Return chiude la porta:** Qualsiasi istruzione scritta al di sotto della parola chiave `return` è considerata codice morto e non verrà mai eseguita dall'interprete.
    
- **Attenzione ai dati complessi:** Ricorda che passare un oggetto o un array a una funzione espone il dato originale a modifiche dirette a causa del passaggio per riferimento.
    

📖 **Glossario**

| **Termine Istituzionale**     | **Definizione Formale**                                                                                                                          | **"Spiega Brutta"**                                                                                                                   |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| **Parametro**                 | Variabile locale specificata nella firma di una funzione per accettare dati in input.                                                            | Il nome-segnaposto che metti tra le parentesi quando progetti la struttura della funzione.                                            |
| **Argomento**                 | Valore effettivo trasferito alla funzione durante la fase di invocazione.                                                                        | Il valore vero e concreto che inserisci tra le parentesi quando chiami la funzione per usarla.                                        |
| **Passaggio per Valore**      | Modalità di trasferimento in cui viene creata una copia fisica del dato nella memoria locale.                                                    | Passare una fotocopia del dato: l'originale fuori dalla funzione non corre rischi.                                                    |
| **Passaggio per Riferimento** | Modalità di trasferimento in cui viene passato l'indirizzo di memoria dell'oggetto originale.                                                    | Passare le chiavi di casa del dato: se cambi un mobile dentro la stanza, cambia anche per il proprietario fuori.                      |
| **Early Exit**                | Pratica di programmazione che consiste nel terminare l'esecuzione di una funzione non appena si verificano le condizioni di errore o di guardia. | Usare il `return` come una via d'uscita d'emergenza per bloccare subito la funzione se qualcosa va storto, senza farle perdere tempo. |
