# 9.8 Postman App (Il banco di prova)

Postman non è un linguaggio di programmazione, ma un software fondamentale per chiunque sviluppi sul web. È un Client HTTP grafico che permette di formulare, configurare e inviare richieste a qualsiasi API senza dover scrivere una singola riga di codice JavaScript. È il tuo laboratorio di analisi privato.

## 🛠️ 9.8.1 Come si mappa la teoria HTTP nell'interfaccia di Postman

Per usare Postman non serve studiare nuove regole, basta applicare i concetti visti nei capitoli precedenti sfruttando le schede della sua interfaccia grafica:

- **La barra dell'URL:** Qui selezioni il metodo dal menu a tendina (GET, POST, ecc.) e incolli l'Endpoint.
    
- **Scheda Params (Query String):** Se scrivi i parametri qui dentro (es. chiave: _categoria_, valore: _scarpe_), Postman costruirà automaticamente la Query String in fondo all'URL inserendo i caratteri `?` e `&` al posto tuo.
    
- **Scheda Headers (Metadati):** Qui inserisci le coppie chiave-valore di servizio, come le chiavi di autenticazione (`Authorization`) o il tipo di dato (`Content-Type`).
    
- **Scheda Body (Carico utile per le POST):** Quando fai una POST, devi andare su _Body_, selezionare l'opzione _raw_ e impostare il formato su **JSON**. Lì dentro potrai scrivere il tuo oggetto JSON rispettando le regole sintattiche.
    

## 🎯 9.8.2 Perché usare Postman invece della console?

Se la tua chiamata `fetch` in JavaScript non funziona, il problema potrebbe risiedere ovunque: un errore di battitura, un problema di asincronia, o magari il server dell'API è spento. Postman serve a isolare il problema:

- **Isolamento dell'errore:** Se la chiamata funziona su Postman ma fallisce nel tuo codice, hai la certezza matematica che il server risponde correttamente e il bug si trova nel tuo JavaScript.
    
- **Le Collezioni (Collections):** Ti permettono di salvare le richieste all'interno di cartelle organizzate per progetto. In questo modo non devi riscrivere l'URL e gli header ogni volta che riapri il programma.
    
- **Generatore di codice integrato:** Postman possiede un pulsante laterale contrassegnato dal simbolo (`</>`). Cliccandolo, lo strumento trasforma la richiesta grafica che hai appena testato in codice JavaScript (`fetch`) puro, formattato e pronto da copiare.
    

## 🔄 9.8.3 Il flusso di lavoro professionale

Ogni volta che devi integrare un'API all'interno di un'applicazione:

1. Apri Postman e crea una richiesta nella cartella (Collection) del tuo progetto.
    
2. Inserisci l'endpoint, seleziona il metodo corretto e imposta i parametri o il Body richiesto.
    
3. Clicca su **Send** e analizza il pannello della risposta: controlla lo Status Code (200 o 201) e verifica nel pannello _Pretty_ che la struttura del JSON sia quella che ti aspetti.
    
4. Solo dopo aver verificato che tutto funzioni alla perfezione, passa all'editor di codice per scrivere la tua funzione `fetch`.

## 🚀 Key Takeaways del Giorno

- **Testa prima, scrivi dopo:** Non sprecare ore a fare il debug del codice JavaScript prima di aver verificato se l'endpoint risponde correttamente.
    
- **Ispezione totale:** Postman mostra dettagliatamente non solo il _body_ della risposta, ma anche il tempo di risposta in ms, la dimensione del file e tutti gli _Header_ restituiti dal server.
    

## 📖 Glossario

| **Termine Istituzionale** | **Definizione Formale**                                                                                | **"Spiega Brutta"**                                                                                        |
| ------------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| **Client HTTP**           | Applicazione software che avvia richieste verso un server HTTP e ne visualizza le risposte.            | Il browser "messo a nudo": uno strumento che fa solo chiamate di rete pure senza grafica o stile.          |
| **Collection**            | Gruppo di richieste HTTP salvate e organizzate in cartelle all'interno di Postman.                     | Il raccoglitore ad anelli: una cartella dove salvi le chiamate di un progetto per non riscriverle da zero. |
| **Sandbox**               | Ambiente software isolato in cui le modifiche al codice o i test possono essere eseguiti in sicurezza. | Il banco di prova: un posto sicuro dove fare esperimenti senza rischiare di rompere il sito reale.         |
