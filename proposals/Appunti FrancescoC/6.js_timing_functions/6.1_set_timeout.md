## ⏱️ 6.1 setTimeout (Il Timer Singolo)

Spesso nello sviluppo software abbiamo bisogno di creare delle azioni ritardate nel tempo. Immagina di voler mostrare un messaggio di avviso all'utente solo dopo che è rimasto sulla pagina per 5 secondi, oppure di voler far sparire una notifica di successo automaticamente dopo un breve momento.

La funzione nativa `setTimeout` serve esattamente a questo: imposta un timer asincrono che attende un determinato periodo di tempo e poi esegue **una sola volta** la funzione di callback che gli abbiamo affidato.

### ⚙️ 6.1.1 La Sintassi Base e il Calcolo del Tempo

Il comando `setTimeout` richiede due informazioni fondamentali per funzionare:

1. **La funzione di Callback** da eseguire alla fine del conto alla rovescia.
    
2. **Il tempo di attesa**, espresso tassativamente in **millisecondi** ($1 \text{ secondo} = 1000 \text{ millisecondi}$).


```JavaScript
// Definizione della funzione di callback in italiano
function mostraMessaggioBenvenuto() {
  console.log("Benvenuto sul sito! Sono passati 3 secondi.");
}

// Avviamo il timer: esegui la funzione dopo 3000 millisecondi (3 secondi)
// NOTA: Passiamo solo il nome della funzione come riferimento, SENZA parentesi tonde!
setTimeout(mostraMessaggioBenvenuto, 3000);

// Variante comune usando una Arrow Function anonima direttamente dentro il timer:
setTimeout(() => {
  console.log("Questo messaggio appare dopo 1 secondo solo.");
}, 1000);
```

### 🛡️ 6.1.2 Annullare un Timer: clearTimeout

Quando attiviamo un `setTimeout`, il browser fa partire un conto alla rovescia invisibile in background. Questo processo genera un codice identificativo univoco (**un ID numerico del timer**).

Se l'utente compie un'azione prima che il tempo sia scaduto, possiamo usare questo ID per bloccare immediatamente il conto alla rovescia ed evitare che la funzione parta, grazie al comando nativo `clearTimeout`.

#### 💻 L'Esempio Pratico e Interattivo

Immagina di avere questo semplice frammento di codice nella tua pagina HTML:

```HTML
<p id="stato-bomba">Conto alla rovescia avviato: 5 secondi all'esplosione...</p>
<button id="btn-disinnesca">Disinnesca Bomba!</button>
```

Ora guarda come gestiamo il blocco del timer nel file JavaScript quando l'utente fa click sul pulsante reale:

```JavaScript
// 1. Selezioniamo gli elementi dal DOM
const testoStato = document.querySelector("#stato-bomba");
const bottoneDisinnesca = document.querySelector("#btn-disinnesca");

// 2. Facciamo partire il timer (salviamo l'ID del timer nella variabile per poterlo bloccare)
let idContatoreBomba = setTimeout(() => {
  testoStato.textContent = "BOOM! La bomba è esplosa! 💥";
  testoStato.style.color = "red";
}, 5000);

// 3. Gestiamo l'evento sul bottone: se l'utente clicca prima di 5 secondi, salviamo la situazione
bottoneDisinnesca.addEventListener("click", () => {
  // Disinneschiamo il timer usando il suo ID univoco memorizzato
  clearTimeout(idContatoreBomba);
  
  // Aggiorniamo la pagina per dare un feedback visivo immediato
  testoStato.textContent = "Timer interrotto con successo. La bomba è sicura! 🛡️";
  testoStato.style.color = "green";
});
```

### 🔀 6.1.3 Passare parametri dentro il setTimeout

Se provi a passare un argomento a una funzione normale dentro il `setTimeout` scrivendo direttamente le parentesi tonde, cadrai nel tranello più comune di JavaScript:

```JavaScript
// ❌ L'ERRORE CLASSICO:
setTimeout(salutaUtente("Filippo"), 2000); // Si rompe! Esegue la funzione subito e non aspetta.
```

Perché succede? Perché le parentesi tonde `()` ordinano a JavaScript di eseguire la funzione immediatamente durante la lettura della riga, anziché lasciarla in attesa.

Per passare i dati correttamente senza far scattare il timer in anticipo, hai due strade professionali:

#### Strada 1: Usare una Arrow Function "guscio" (Scelta Consigliata)

Invece di lanciare la funzione direttamente, la impacchetti dentro una funzione a freccia vuota `() =>`. Il `setTimeout` vede la freccia e si mette in attesa del tempo prestabilito. Solo quando il timer scade, la freccia si apre ed esegue il codice al suo interno.

```JavaScript
// Il timer vede la freccia e aspetta. Dopo 2 secondi apre il guscio ed esegue il saluto
setTimeout(() => salutaUtente("Filippo"), 2000); 
```

#### Strada 2: Sfruttare i parametri successivi di setTimeout

I creatori del linguaggio hanno inserito una scorciatoia nativa: se inserisci dei dati dopo aver impostato il tempo di attesa, il browser li prenderà e li passerà come argomenti alla tua funzione quando scatta il timer.

```JavaScript
// Struttura: setTimeout(funzione, tempoInMillisecondi, parametro1, parametro2...)
setTimeout(salutaUtente, 2000, "Filippo"); 
```

#### 📊 Tabella di Confronto: Quale approccio scegliere?

|**Soluzione**|**Sintassi**|**Quando usarla?**|**Perché è comoda?**|
|---|---|---|---|
|**Arrow Function**|`() => funzione(dati)`|Nel 90% dei casi reali|È visiva, moderna e facilissima da leggere se devi passare più dati o fare calcoli al volo nello script.|
|**Parametri Successivi**|`funzione, tempo, dati`|Casi molto semplici|Non devi creare funzioni guscio extra, anche se con molti parametri il codice può risultare meno immediato da scorrere.|

### ⚠️ 6.1.4 Errori comuni

- **Inserire le parentesi tonde di esecuzione nella callback:** Scrivere `setTimeout(mostraMessaggio(), 2000)` farà eseguire la funzione immediatamente all'avvio dell'applicazione, ignorando il ritardo. Passa sempre e solo il **nome di riferimento** della funzione.
    
- **Sbagliare la conversione dei millisecondi:** Ricorda sempre che il motore del browser ragiona in millisecondi. Se vuoi un ritardo di 5 secondi e imposti semplicemente il numero `5`, il timer durerà 5 millisecondi (un battito di ciglia impercettibile), eseguendo l'azione praticamente all'istante.

🚀 **Key Takeaways del Giorno**

- **`setTimeout` è un timer usa e getta:** Esegue la sua azione programmata una volta sola e poi distrugge il suo ciclo di vita autonomamente.
    
- **Il tempo viaggia in millisecondi:** Moltiplica sempre i secondi che desideri ottenere per 1000 quando imposti il valore del timer.
    
- **I timer si possono bloccare:** Conserva sempre l'ID numerico restituito da `setTimeout` all'interno di una variabile se prevedi di dover annullare il conto alla rovescia tramite `clearTimeout`.
