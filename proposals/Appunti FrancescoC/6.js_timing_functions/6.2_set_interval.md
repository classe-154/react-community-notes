## 🔄 6.2 setInterval (Il Timer Ciclico)

Mentre `setTimeout` funziona come una sveglia singola che suona una volta sola e poi si spegne, `setInterval` funziona come un metronomo o un timer a ripetizione.

Questo comando dice a JavaScript di eseguire una determinata funzione di callback ripetutamente, all'infinito, ogni volta che scade un preciso intervallo di tempo. È lo strumento perfetto quando dobbiamo creare elementi che si aggiornano continuamente, come un orologio digitale sul sito, un conto alla rovescia visivo (countdown), o un'animazione ripetuta.

### ⚙️ 6.2.1 La Sintassi e il Ciclo Infinito

La sintassi è identica a quella del `setTimeout`. Richiede anch'essa una funzione da eseguire e il tempo di attesa espresso in millisecondi.

```JavaScript
// Funzione che simula il battito di un orologio
function aggiornaOrologio() {
  console.log("Tic... è passato un altro secondo.");
}

// Avviamo l'intervallo: esegui la funzione OGNI secondo (1000 millisecondi)
// NOTA: Passiamo il nome della funzione senza parentesi tonde!
setInterval(aggiornaOrologio, 1000);
```

Se lanci questo codice, il browser continuerà a stampare in console "Tic..." all'infinito, finché l'utente non chiude la scheda del sito.

### 🛑 6.2.2 Fermare la Ripetizione: clearInterval

Proprio perché `setInterval` non si ferma mai da solo, abbiamo bisogno di un modo ingegneristico per bloccarlo quando l'azione è terminata.

Quando facciamo partire un `setInterval`, il browser ci restituisce un codice identificativo univoco (un ID numerico). Passando questo ID dentro il comando `clearInterval`, il metronomo si stoppa all'istante.

#### 💻 L'Esempio Pratico e Interattivo (Evitare l'accumulo)

Colleghiamo un countdown direttamente a un'interfaccia HTML per vedere come gestire lo spegnimento e impedire all'utente di cliccare più volte facendo impazzire il timer.

**HTML:**
```HTML
<h1 id="visualizzatore-timer">5</h1>
<button id="btn-avvia">Avvia Countdown</button>
```

**JAVASCRIPT:**
```JavaScript
// Nel file script.js
const displayTempo = document.querySelector("#visualizzatore-timer");
const bottoneAvvia = document.querySelector("#btn-avvia");

let secondiRimanenti = 5;
let idIntervallo = null; // Scatola vuota per salvare l'ID del timer

bottoneAvvia.addEventListener("click", () => {
  // ⚠️ SOLUZIONE ALL'ACCUMULO: Disabilitiamo il bottone così l'utente non può cliccare di nuovo
  bottoneAvvia.disabled = true;
  
  // Ripristiniamo i secondi se il timer era già stato usato in precedenza
  secondiRimanenti = 5; 
  displayTempo.textContent = secondiRimanenti;

  // 1. Avviamo il metronomo e salviamo il suo ID
  idIntervallo = setInterval(() => {
    secondiRimanenti--;
    displayTempo.textContent = secondiRimanenti; // Aggiorna lo schermo ad ogni secondo
    
    // Condizione di uscita naturale
    if (secondiRimanenti === 0) {
      // 2. Blocchiamo il timer quando arriva a zero
      clearInterval(idIntervallo);
      
      displayTempo.textContent = "PARTENZA! 🚀";
      bottoneAvvia.disabled = false; // Riattiviamo il bottone per un futuro riutilizzo
    }
  }, 1000);
});
```

### ⚠️ 6.2.3 Errori comuni

- **Creare intervalli infiniti senza una via d'uscita:** Se avvii un `setInterval` e dimentichi di salvare il suo ID per fare il `clearInterval`, l'applicazione continuerà a eseguire quella funzione in sottofondo per sempre. Questo consuma la memoria del computer dell'utente (_Memory Leak_) e rallenta tutto il browser.
    
- **Accumulare timer identici:** Se inserisci un `setInterval` dentro il click di un bottone, ogni volta che l'utente clicca sul bottone partirà un nuovo timer che si aggiunge a quelli vecchi. La console inizierà a impazzire stampando i messaggi a doppia o tripla velocità. Prima di far partire un timer, assicurati sempre che quello vecchio sia stato spento.

🚀 **Key Takeaways del Giorno**

- **`setInterval` non si stanca mai:** Continua a ripetere la funzione a intervalli regolari finché non riceve un ordine esplicito di stop.
    
- **Pensa sempre allo spegnimento:** Ogni volta che scrivi un `setInterval`, devi avere in mente la condizione logica o il pulsante che attiverà il `clearInterval`.
    
- **Lavora in isolamento:** Salva sempre l'ID del timer in una variabile dedicata per avere il controllo totale del suo ciclo di vita.
    

📖 **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                                               | **"Spiega Brutta"**                                                                                                      |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **`setInterval`**         | Metodo che chiama ripetutamente una funzione o esegue un frammento di codice, con un ritardo fisso tra ogni chiamata. | Un ripetitore automatico: esegue un blocco di codice a ritmo di metronomo ogni tot millisecondi.                         |
| **`clearInterval`**       | Funzione nativa che cancella un'azione ripetitiva programmata a tempo impostata tramite `setInterval`.                | Il tasto "Stop" del lettore musicale: blocca immediatamente la ripetizione infinita del timer.                           |
| **Memory Leak**           | Consumo non intenzionale di memoria RAM dovuto a risorse non più necessarie che non vengono rilasciate dal programma. | Spreco di memoria: quando lasci un rubinetto del codice aperto (come un timer infinito) che consuma risorse di nascosto. |
