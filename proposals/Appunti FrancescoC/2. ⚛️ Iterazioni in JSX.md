# 2.⚛️ Iterazioni in JSX

### 2.1 Metodo .map() 
per creare liste In JSX non puoi usare cicli tradizionali come `for` o `while` dentro il `return`. Per ripetere elementi (come una lista di nomi), si usa il metodo degli array `.map()`, che trasforma ogni dato in un pezzetto di interfaccia.

- **Azione:** Ogni elemento dell'array viene "mappato" in un tag (es. `<li>`).
    
- **Esempio:**
    
```js
const nomi = ["Luca", "Maria", "Giovanni"];

function Lista() {
  return (
    <ul>
      {nomi.map((nome) => (
        <li key={nome}>{nome}</li>
      ))}
    </ul>
  );
}
```

Quando usi `.map()`, spesso vedrai due modi di scrivere la freccia `=>`. È una delle fonti principali di bug per i principianti:

 **Parentesi Tonde `()` (Return Implicito):** Usale se vuoi restituire direttamente il JSX. È la più pulita.
 
```JavaScript
   {nomi.map(nome => (
       <li>{nome}</li> // Restituisce automaticamente
   ))} 
```


 **Parentesi Graffe `{ }` (Corpo della funzione):** Se le usi, **DEVI** scrivere esplicitamente la parola `return`, altrimenti il componente non mostrerà nulla (restituirà `undefined`).

```JavaScript
   {nomi.map(nome => {
   const nomeMaiuscolo = nome.toUpperCase();
   return <li>{nomeMaiuscolo}</li>; // Il return è obbligatorio qui!
})}
```

#### 2.1.1 Il mistero della prop `key`
Quando crei una lista, React pretende che ogni elemento abbia una `key` unica. Serve al "motore" di React per capire esattamente quale pezzo della lista è cambiato, evitando di rinfrescare tutto inutilmente.

- **Pro Tip:** La `key` deve essere stabile. Usa l'ID del database se ce l'hai. Evita l'indice (`index`) del ciclo se la lista può essere ordinata o filtrata, altrimenti React farà confusione.
    

#### 2.1.2 Filtri e Logica Applicata 
Puoi pulire i dati prima di mostrarli usando `.filter()` o inserire condizioni veloci (operatore ternario) dentro il ciclo.

- **Esempio con Filtro:** Mostra solo i nomi che iniziano con "M".
    

```js
{nomi
  .filter((nome) => nome.startsWith("M"))
  .map((nome) => <li key={nome}>{nome}</li>)
}
```

- **Pro Tip: Concatenazione (Chaining)**

In React è comunissimo vedere `.filter()` e `.map()` attaccati. Ricorda che **l'ordine conta**: prima filtri la lista (ne riduci il numero) e poi mappi il risultato in JSX.

```js
<ul>
  {prodotti
    .filter(p => p.prezzo < 50)           // 1. Prendi solo quelli economici
    .map(p => <li key={p.id}>{p.nome}</li>) // 2. Trasformali in <li>
  }
</ul>
```
## 2.2 Strategie di Pulizia (Organizzazione)

#### 2.2.1 Variabili e Funzioni di Supporto
Se il ciclo `.map()` diventa troppo lungo e sporca il `return`, puoi spostare la logica fuori.

- **Scenario di partenza

```js
	const prodotti = [
  { id: 1, nome: "Smartphone", prezzo: 500 },
  { id: 2, nome: "Laptop", prezzo: 1200 },
  { id: 3, nome: "Mouse", prezzo: 25 }
];
```

- **Tecnica 1:** Salva la lista in una variabile prima del `return`.

	- In questo caso, esegui il `.map()` **sopra** il `return` del componente. Questo rende il JSX finale estremamente pulito, perché dovrai solo richiamare la variabile tra parentesi graffe.

```js
	function ListaProdotti() {
  // Elaboriamo la logica fuori dal return
  const elementiLista = prodotti.map((prodotto) => (
    <li key={prodotto.id}>
      <strong>{prodotto.nome}</strong>: {prodotto.prezzo}€
    </li>
  ));

  return (
    <div>
      <h2>I nostri Prodotti</h2>
      {/* Qui il template rimane pulitissimo */}
      <ul>{elementiLista}</ul>
    </div>
  );
}
```
    
- **Tecnica 2:** Crea una funzione `renderLista()` che restituisce il JSX e chiamala nel template. Questo mantiene il componente principale leggibile e ordinato.

	- Questa tecnica è ottima quando la logica di rendering è complessa o dipende da alcune condizioni (ad esempio, se vuoi mostrare un messaggio diverso se la lista è vuota).

```js
function ListaProdotti() {
  
  // Creiamo una funzione interna (helper function)
  const renderLista = () => {
    if (prodotti.length === 0) {
      return <p>Nessun prodotto disponibile.</p>;
    }

    return prodotti.map((prodotto) => (
      <li key={prodotto.id}>
        {prodotto.nome} - {prodotto.prezzo}€
      </li>
    ));
  };

  return (
    <div>
      <h2>Catalogo Online</h2>
      <ul>
        {/* Chiamiamo la funzione nel template */}
        {renderLista()}
      </ul>
    </div>
  );
}
```

### 🚀 Key Takeaways del Giorno

- **No Cicli Classici dopo il return:** Dimentica il `for` dentro il JSX; usa `.map()`.
    
- **Key Obbligatoria:** Senza una chiave unica, React ti lancerà un errore in console e le performance caleranno.
    
- **Concatenazione:** Puoi usare `.filter().map()` per mostrare solo quello che serve in un colpo solo.
    
- **Ordine:** Se il codice JSX diventa un groviglio, sposta la logica delle liste in variabili esterne.

### 📖 Glossario

| **Termine Istituzionale** | **Definizione Formale**                                           | **"Spiega Brutta"**                                                                                                  |
| ------------------------- | ----------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **.map()**                | Metodo che crea un nuovo array manipolando ogni elemento.         | Trasforma ogni elemento di una lista "grezza" (es. nomi) in un pezzo di interfaccia (es. `<li>`).                    |
| **Key**                   | Attributo speciale per identificare elementi in una lista.        | Un'etichetta identificativa per non far confondere React quando aggiungi o togli pezzi.                              |
| **.filter()**             | Metodo che crea un array con i soli elementi che passano un test. | Un buttafuori che decide chi può entrare nella lista e chi resta fuori.                                              |
| **Reconciliation**        | Algoritmo di React per aggiornare il DOM.                         | Il processo con cui React decide cosa "ridipingere" nel browser, evitando di rifare tutto da capo grazie alle `key`. |
