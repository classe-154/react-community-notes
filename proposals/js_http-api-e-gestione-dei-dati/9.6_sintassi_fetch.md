## ⚡ 9.6 Sintassi Fetch (Il cuore delle chiamate)

La `fetch` è la funzione nativa di JavaScript che ci permette di effettuare chiamate HTTP verso un server. È lo strumento moderno e standard che sostituisce le vecchie tecniche complesse (come `XMLHttpRequest`).

### 🧱 9.6.1 La struttura di Fetch e il meccanismo delle Promise

La funzione `fetch()` restituisce sempre una **Promise**. Una Promise è un oggetto che rappresenta un'operazione asincrona non ancora conclusa (un "risultato futuro").

Il modo standard di gestire una Promise è tramite la catena di `.then()` e `.catch()`:

```JavaScript
// La forma base di una chiamata fetch (di default è un metodo GET)
fetch('https://api.esempio.it/v1/prodotti')
  .then((response) => {
    // TRAPPOLA TECNICA: Controlliamo se il server ha risposto con un errore (es. 404 o 500)
    if (!response.ok) {
      // Lanciamo un errore manualmente per saltare direttamente al .catch()
      throw new Error(`Errore del server: ${response.status}`);
    }
    
    // Se tutto è OK, convertiamo la risposta testuale in un oggetto JS
    return response.json(); 
  })
  .then((dati) => {
    // Qui i dati sono pronti, puliti e utilizzabili dentro la memoria
    console.log("Dati ricevuti con successo:", dati);
  })
  .catch((errore) => {
    // Gestisce sia i crash di rete sia gli errori lanciati con "throw new Error"
    console.error("Si è verificato un problema nella chiamata:", errore.message);
  });
```

### 🔄 9.6.2 Perché servono due fasi (`.then()`)?

La conversione richiede due passaggi separati perché la risposta viaggia sulla rete in momenti diversi:

1. **Primo `.then()`:** Riceve l'oggetto `response` contenente solo i metadati (status code e header). Qui verifichiamo se lo status è corretto e diciamo al browser di avviare lo scaricamento e la conversione del testo con `response.json()`.
    
2. **Secondo `.then()`:** Poiché la conversione del testo in oggetto JavaScript richiede tempo, `response.json()` genera una seconda Promise. Questo secondo blocco si attiva solo quando l'oggetto JavaScript è stato interamente ricostruito in memoria.
    

### ⚠️ Reminder Tecnico & Best Practice

- **La trappola del finto successo:** `fetch()` **non** fallisce se il server restituisce `404` o `500`. Per la fetch, se il server risponde, la richiesta è riuscita. Devi controllare `response.ok` e lanciare un errore manualmente per far scattare il `.catch()`.
    
- **Configurazione per l'invio dati (POST):** Se devi inviare dati, passa un secondo argomento (oggetto di configurazione) per impostare il metodo, gli header e trasformare il tuo oggetto in stringa JSON:
    
    ```    JavaScript
    fetch('https://api.esempio.it/v1/prodotti', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(nuovoProdotto)
    })
    .then(response => response.json())
    .then(dati => console.log("Prodotto salvato:", dati));
    ```

🚀 **Key Takeaways del Giorno**

- **Asincronia non bloccante:** Il codice sotto la `fetch` prosegue mentre la rete lavora. Non puoi usare i dati della `fetch` al di fuori dei suoi `.then()`.
    
- **response.ok è obbligatorio:** La gestione degli errori HTTP richiede un controllo manuale nel primo step.
    
- **Catena logica:** Prima si chiama l'indirizzo, poi si verifica/converte il testo, infine si usano i dati reali.
