## ⚙️ 1. **Le Variabili (let e const)**

In JavaScript moderno, abbiamo due modi principali per creare le nostre variabili, a seconda di come vogliamo gestire il loro contenuto nel tempo: `let` e `const`.

- **let** si usa quando il contenuto della variabile può cambiare nel tempo. Ad esempio, il punteggio di un videogioco, il numero di prodotti nel carrello della spesa o i secondi di un timer.
    
- **const** (abbreviazione di costante) si usa quando il contenuto della variabile non deve mai cambiare per tutta la durata del programma. Si usa per dati fissi, come l'anno di nascita di un utente, la formula della gravità o il nome di una cartella di sistema.
    

## 💻 2. **Esempio base**

Vediamo come si creano e come si comportano queste due variabili in un programma elementare.

```js
// Usiamo 'let' perché il punteggio cambierà durante la partita
let punteggio = 0;
punteggio = 10; // Aggiorniamo il valore: adesso dentro la variabile c'è 10

// Usiamo 'const' perché la data di nascita è fissa e non cambia mai
const annoNascita = 2000;

console.log(punteggio);
console.log(annoNascita);
```

🔍 **Spiegazione dell'esempio:**

- `let punteggio = 0;`: Creiamo una variabile modificabile chiamata punteggio e ci mettiamo dentro lo zero. (dichiarazione e assegnazione in una linea, inizializzazione)
    
- `punteggio = 10;`: Decidiamo di cambiare il valore. Noti qualcosa? Non abbiamo riscritto la parola let. La parola chiave serve solo la prima volta per dichiarare la variabile. Per cambiare il contenuto in seguito, basta usare il nome della variabile e il simbolo = (assegnazione o riassegnazione in questo caso). Ora lo zero è stato rimpiazzato dal 10.
    
- `const annoNascita = 2000;`: Creiamo una variabile bloccata chiamata annoNascita. Se provassimo a scrivere `annoNascita = 2005;`, JavaScript bloccherà il programma con un errore rosso, dicendoci che non possiamo cambiare una costante.
    
## 🛠️ 3. **Esempio pratico**

Immaginiamo di gestire un negozio online. Dobbiamo calcolare il prezzo finale di un prodotto applicando uno sconto fisso.

```js
// Il prezzo del prodotto può cambiare se l'utente aggiunge opzioni
let prezzoIniziale = 50; 

// Lo sconto del negozio è fisso per oggi, non deve essere modificato per errore
const scontoEuro = 5;

// Calcoliamo il prezzo finale sottraendo lo sconto
let prezzoFinale = prezzoIniziale - scontoEuro;

console.log("Prezzo da pagare: " + prezzoFinale + " Euro");
```

🔍 **Spiegazione dell'esempio:**

- Il programma crea la variabile modificabile prezzoIniziale con dentro il numero 50 e una variabile bloccata scontoEuro con dentro il numero 5.
    
- Successivamente crea una terza variabile chiamata prezzoFinale. Per decidere cosa metterci dentro, JavaScript fa un'operazione matematica: va a leggere dentro prezzoIniziale (50), va a leggere dentro scontoEuro (5) e fa la sottrazione 50 - 5.
    
- Il risultato (45) viene salvato dentro prezzoFinale e stampato in console.
    

## ⚠️ 4. **Errori comuni**

- **Provare a modificare una costante (const):** Scrivere `const nome = "Luca"; nome = "Marco";` blocca il programma. Se un dato deve cambiare nel tempo, usa sempre let.
    
- **Riscrivere la parola chiave quando si cambia valore:** Scrivere `let contatore = 1; let contatore = 2;` provoca un errore. JavaScript penserà che stai cercando di creare due variabili diverse con lo stesso nome nella stessa stanza. Per aggiornare il valore, non ripetere let.
    
- **Creare una const vuota:** Scrivere `const codiceSegreto;` senza assegnargli subito un valore causerà un SyntaxError. Le costanti devono essere inizializzate immediatamente perché non possono essere cambiate in seguito.
    
## 💡 5. **Approfondimento leggero: Best practice nella scelta tra Let e Const**

Nello sviluppo software professionale moderno, la regola d'oro è: **usa const come scelta predefinita.**

Quando crei una nuova variabile, dichiarala sempre con `const`. Solo nel momento in cui ti rendi conto che quel valore avrà bisogno di essere sovrascritto o modificato nel corso del codice, cambiala in `let`. Questo approccio rende il codice molto più sicuro, perché impedisce modifiche accidentali ai dati.

## 7. **Key Takeaways del Giorno**

- **Usa let per il cambiamento:** Scegli let solo se sai con certezza che il valore della variabile dovrà cambiare o essere aggiornato durante l’esecuzione del programma.
    
- **Usa const come default:** Utilizzare le costanti come scelta predefinita riduce i bug e rende il codice più robusto, ordinato e sicuro, oltre che più leggibile (chiunque guardi il codice sa che se vede una variabile dichiarata con const, questa variabile non cambierà nel tempo, ad esclusione di alcuni casi molto particolari).
    
- **Dichiarazione unica:** Le parole chiave let e const si usano solo la prima volta che si crea la variabile, mai quando se ne aggiorna il valore.
    

## 8. **Glossario**

| **Termine Istituzionale** | **Definizione Formale**                                                                            | **"Spiega Brutta"**                                                                                 |
| ------------------------- | -------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| **Costante**              | Variabile il cui valore non può essere riassegnato durante l'esecuzione del programma.             | Una variabile creata con const che viene sigillata e non può più cambiare contenuto.                |
| **Riassegnazione**        | Operazione con cui si sostituisce il valore precedentemente memorizzato in una variabile.          | Il processo di cambiare il valore dentro una variabile usando il simbolo =, possibile solo con let. |
| **Inizializzazione**      | L'atto di assegnare un valore iniziale a una variabile nel momento stesso in cui viene dichiarata. | Riempire subito la variabile di dati appena la si crea (obbligatorio per le const).                 |
